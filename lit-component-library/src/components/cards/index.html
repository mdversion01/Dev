Props
Property Type Default Description
actions Boolean false When this is true, the user can add '<div slot="actions">Action Buttons or Links can go here.</div>' to provide the card with any action or interactive elements and wraps the actions in a div with the class of 'pl-card-actions'.
altText String When adding an image to the top of the card, use this property to set the image's 'alt' text.
Ex: altText="Image Alt Text"
cardMaxWidth String 20 This property allows the user to set the 'max-width' of the card in rem's not px's. The style is place inline of the 'pl-card' div and the default is '20rem'.
Ex: cardMaxWidth="10"
img Boolean false When this is true, an area is provided for the user to adda n image.
imgSrc String When the 'img' property is true, use this property to provide the path or url to an image.
inlineStyles String When using this property, additional inline styles can be added to the card.
Ex:<card-component inlineStyles="color: pink; padding: 10px;"></card-component>
noFooter Boolean false When this is true, the footer is not displayed and the slotted content will not be shown.
noHeader Boolean false When this is true, the header is not displayed and the slotted content will not be shown.
Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

In a Vue component, you'd use CardComponent just like any other HTML element: 

          
            <template>
                <div>
                    <lit-element-wrapper></lit-element-wrapper>
                </div>
            </template>
              
            <script>
              import { LitElement, html, css } from 'lit';
              import { property, customElement } from 'lit/decorators.js';
              
              @customElement('lit-element-wrapper')
              class LitElementWrapper extends LitElement {
                @property({ type: Boolean }) actions = true;
                @property({ type: String }) altText = 'Sample Image';
                @property({ type: String }) cardWidth = '300px';
                @property({ type: String }) classNames = 'custom-class';
                @property({ type: Boolean }) img = true;
                @property({ type: String }) imgSrc = 'path/to/image.jpg';
                @property({ type: String }) inlineStyles = 'color: red;';
                @property({ type: Boolean }) noFooter = false;
                @property({ type: Boolean }) noHeader = false;
              
                render() {
                  return html`
                    <card-component
                      .actions="${this.actions}"
                      .altText="${this.altText}"
                      .cardWidth="${this.cardWidth}"
                      .classNames="${this.classNames}"
                      .img="${this.img}"
                      .imgSrc="${this.imgSrc}"
                      .inlineStyles="${this.inlineStyles}"
                      .noFooter="${this.noFooter}"
                      .noHeader="${this.noHeader}"
                    >
                    <div slot="header">Header Content</div>
                    <div slot="title">Title Content</div>
                    <div slot="text">Text Content</div>
                    <div slot="actions">Actions Content</div>
                    <div slot="footer">Footer Content</div>
                    </card-component>
                  `;
                }
              }
              
              export default {
                components: {
                  LitElementWrapper,
                },
              };
              </script>
React

In a React component, you'd similarly treat CardComponent as a regular HTML element: 

          
            import React from 'react';
            import { LitElement, html, css } from 'lit';
            import { property, customElement } from 'lit/decorators.js';

            @customElement('lit-element-wrapper')
            class LitElementWrapper extends LitElement {
            @property({ type: Boolean }) actions = true;
            @property({ type: String }) altText = 'Sample Image';
            @property({ type: String }) cardWidth = '300px';
            @property({ type: String }) classNames = 'custom-class';
            @property({ type: Boolean }) img = true;
            @property({ type: String }) imgSrc = 'path/to/image.jpg';
            @property({ type: String }) inlineStyles = 'color: red;';
            @property({ type: Boolean }) noFooter = false;
            @property({ type: Boolean }) noHeader = false;

            render() {
                return html`
                <card-component
                    .actions="${this.actions}"
                    .altText="${this.altText}"
                    .cardWidth="${this.cardWidth}"
                    .classNames="${this.classNames}"
                    .img="${this.img}"
                    .imgSrc="${this.imgSrc}"
                    .inlineStyles="${this.inlineStyles}"
                    .noFooter="${this.noFooter}"
                    .noHeader="${this.noHeader}"
                >
                    <div slot="header">Header Content</div>
                    <div slot="title">Title Content</div>
                    <div slot="text">Text Content</div>
                    <div slot="actions">Actions Content</div>
                    <div slot="footer">Footer Content</div>
                </card-component>
                `;
            }
            }

            function App() {
            return (
                <div>
                <LitElementWrapper />
                </div>
            );
            }

            export default App;
Angular

In Angular, you'd use CardComponent as part of your Angular template:

          
            import { Component } from '@angular/core';
            import { LitElement, html, css } from 'lit';
            import { property, customElement } from 'lit/decorators.js';

            @customElement('lit-element-wrapper')
            class LitElementWrapper extends LitElement {
            @property({ type: Boolean }) actions = true;
            @property({ type: String }) altText = 'Sample Image';
            @property({ type: String }) cardWidth = '300px';
            @property({ type: String }) classNames = 'custom-class';
            @property({ type: Boolean }) img = true;
            @property({ type: String }) imgSrc = 'path/to/image.jpg';
            @property({ type: String }) inlineStyles = 'color: red;';
            @property({ type: Boolean }) noFooter = false;
            @property({ type: Boolean }) noHeader = false;

            render() {
                return html`
                <card-component
                    .actions="${this.actions}"
                    .altText="${this.altText}"
                    .cardWidth="${this.cardWidth}"
                    .classNames="${this.classNames}"
                    .img="${this.img}"
                    .imgSrc="${this.imgSrc}"
                    .inlineStyles="${this.inlineStyles}"
                    .noFooter="${this.noFooter}"
                    .noHeader="${this.noHeader}"
                >
                    <div slot="header">Header Content</div>
                    <div slot="title">Title Content</div>
                    <div slot="text">Text Content</div>
                    <div slot="actions">Actions Content</div>
                    <div slot="footer">Footer Content</div>
                </card-component>
                `;
            }
            }

            @Component({
            selector: 'app-root',
            template: `
                <div>
                <lit-element-wrapper></lit-element-wrapper>
                </div>
            `,
            })
            export class AppComponent {}

These examples assume that you've configured your framework project to work with web components and that you've imported the Lit Element component correctly. Adjust import paths accordingly.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
