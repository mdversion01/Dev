Basic Usage
Badge dots are used as indicators to draw attention to a specific element. Used here with the button and icon components.

Props

Property Type Default Description
absolute Boolean false Applies the inline style of position: absolute to the component.
bdgPosition String left When placing a badge on the left of any content, this adds the class 'mr-1' to the badge, which provide 4px's of space between the badge and the content. Placing the badge to the right, and assigning 'right' to the propery will do the reverse, adding the class of 'ml-1'.
Ex: bdgPosition="left" or bdgPosition="right"
bordered Boolean false This property will add a border around a solid colored badge token, the border color will be inherit from the color of the font. This is no the same as the 'outlined' property.
bottom String number | string Positions the component towards the bottom using the inline style attribute. This should be used with the absolute prop.
Ex: bottom="50"
color String This property lets the user add an inline style for the css 'color' element. The hex color, rgb, rgba, and hsl values can be used.
disabled Boolean false This property adds disabled to the button element.
dot Boolean false This property applies a dot indicator(used with a color) to draw attention to a specific element.
icon Boolean false When using an icon with the badge, this property will add the some spacing to the left and right of the icon.
inlineStyles String When using this property, additional inline styles can be added to the badge.
Ex:<pl-badge inlineStyles="margin: 5px;"></pl-badge>
inset Boolean false This property is to be used with the badge tokens. When used it adds the set default style of 'inset: auto auto calc(100% - 12px) calc(100% - 12px);' that positions the badge token to the upper right of the element it is used with.
left String number | string Positions the component towards the left using the inline style attribute. This should be used with the absolute prop.
Ex: left="50"
offsetX String number | string This property is used with the 'inset' property if the user wants to overwrite the default bottom position value of the 'inset'.
Ex: offsetX="50"
offsetY String number | string This property is used with the 'inset' property if the user wants to overwrite the default left position value of the 'inset'.
Ex: offsetY="50"
outlined Boolean false When added, this makes the background transparent and applies a thin border around the badge, unless adding it to the token, then the token's outlined background will be white.
right String number | string Positions the component towards the right using the inline style attribute. This should be used with the absolute prop.
Ex: right="50"
shape String This property allows the user to change the shape of the badge to square, pill, rounded(default) for badge and badge tokens, and circle for badge tokens.
Ex: shape="square"
size String This property allows the user to change the size of the badge to xs(x-small), sm(small), default and lg(large).
Ex: size="sm"
styles String When using this property, additional inline styles can be added to the dot or token.
Ex:<pl-badge styles="color: pink; padding: 10px;"></pl-badge>
token Boolean false This property changes a badge into a badge token. A badge token, by default is placed to the upper right on which it is used. Commonly used with buttons, icons, and tabs. The position can be changed using the offsetX and offsetY properties.
top String number | string Positions the component towards the top using the inline style attribute. This should be used with the absolute prop.
Ex: top="50"
variant String This property allows the user to change the color of the badge to one of the predefined colors/styles.
Ex: varian="primary"
zIndex String number | string Specifies the stack order of an element. This should be used with the absolute or fixed props.
Ex: zIndex="50"

Styling

These class names can be added directly to the component similiar to Props.

Style Class Name Description
Color primary, secondary, tertiary, active-blue, success, danger, warning, light, dark, blue, blue-grey, brown, deep-orange, deep-purple, green, grey, indigo, light-blue, pink, purple, red, teal, shades black, shades white, amber, cyan, lime, light-green, orange, yellow These class names can be added to the component by using the 'variant' property (shown above in Props) and will set background color, and border color for a solid badge.
Ex: <pl-badge variant="primary">Badge Name</pl-badge>
When using these class names with the 'variant' and 'outlined' prop, the background color is transparent and it sets the border color and text color.
Ex: <pl-badge variant="primary" outlined>Badge Name</pl-badge>
Elevation flat, is-elevated, elevated-0 to elevated-24 The elevation helpers allow you to control relative depth, or distance, between two surfaces. There is a total of 25 elevation levels. You can set an element’s elevation by using the class elevated-{n}, where n is a integer between 0-24 corresponding to the desired elevation. Applying flat will give give the button no depth and applying is-elevated will give the button a slight amount of depth..
Ex: <pl-badge variant="primary" flat >Badge Name</pl-badge>
Ex: <pl-badge variant="primary" is-elevated >Badge Name</pl-badge>
Ex: <pl-badge variant="primary" elevated-3 >Badge Name</pl-badge>
Sizes xs, sm, default, lg Sizing for text buttons. 'size="xs"' is 20px, 'size="sm"' is 28px, 'size="default"' is 36px and 'size="lg"' is 44px.
Ex: <pl-badge variant="primary" size="xs" >Badge Name</pl-badge>
Ex: <pl-badge variant="primary" size="sm" >Badge Name</pl-badge>
Ex: <pl-badge variant="primary" size="default" >Badge Name</pl-badge>
Ex: <pl-badge variant="primary" size="lg" >Badge Name</pl-badge>
Shape circle, pill, rounded, square By default, badge and badge tokens, have rounded corners, in which the user does not have to use the 'shape' prop but, they can use the 'rounded' value to be sure that the corners if they so desire. Badges also use the 'square' and the 'pill' for squared and pill shaped badges. Badge tokens can use those three and also the 'circle' value to make the token a circle.
Ex: <pl-badge variant="primary" shape="circle" icon="fas fa-home" ></pl-badge> *This is used with badge tokens.
Ex: <pl-badge variant="primary" shape="pill" >Badge Name</pl-badge>
Ex: <pl-badge variant="primary" shape="square" >Badge Name</pl-badge>
Ex: <pl-badge variant="primary" shape="rounded" >Badge Name</pl-badge>

Using Component in other JavaScript Frameworks

Vue.js

In a Vue component, you'd use PlBadge just like any other HTML element: 

        
          <template>
          <div>
          <!-- Vue component using the PlBadge component -->
          <pl-badge
                bordered
                variant="red"
                dot
                :icon="true"
                :token="false"
                :disabled="false"
                >
              <!-- Content within the badge -->
              <span slot="icon">V</span>
              </pl-badge>
              </div>
          </template>
          
          <script>
          // Import the PlBadge component
          import 'path/to/PlBadge.js'; // Adjust the path to your PlBadge file
          
          export default {
            name: 'VueBadgeComponent',
            // Other Vue component properties and methods
          };
          </script>
React

In a React component, you'd similarly treat PlBadge as a regular HTML element: 

        
          import React from 'react';
          // Import the PlBadge component
          import 'path/to/PlBadge.js'; // Adjust the path to your PlBadge file

          const ReactBadgeComponent = () => {
            return (
              <div>
                {/* React component using the PlBadge component */}
                <pl-badge
                  bordered
                  variant="blue"
                  dot
                  icon={true}
                  token={false}
                  disabled={false}
                >
                  {/* Content within the badge */}
                  <span slot="icon">R</span>
                  </pl-badge>
                </div>
            );
          };

          export default ReactBadgeComponent;
Angular

In Angular, you'd use PlBadge as part of your Angular template:

        
          import { Component } from '@angular/core';

          // Register the PlBadge component in your Angular module
          // Import the PlBadge component in your Angular module file
          import 'path/to/PlBadge.js'; // Adjust the path to your PlBadge file

          @Component({
            selector: 'app-angular-badge',
            template: `
            <div>
            <!-- Angular component using the PlBadge component -->
                <pl-badge
                  bordered
                  [variant]="'green'"
                  [dot]="true"
                  [icon]="true"
                  [token]="false"
                  [disabled]="false"
                >
                <!-- Content within the badge -->
                <span slot="icon">A</span>
                </pl-badge>
              </div>
            `,
          })
          export class AngularBadgeComponent {
            // Angular component properties and methods
          }

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
