Props
Styling
Property Type Default Description
additionalHeaderContent String Additional content to be added to the header of the toast. An example of this would be "43 seconds ago" in the Toast examples above.
appendToast Boolean false Appends the toast to the end of the toast stack. This is useful for Plumage styled toasts.
bodyClass String Adds a class to the body of the toast.
duration Number 5000 The duration in milliseconds that the toast will be displayed.
headerClass String Adds a class to the header of the toast.
iconPlumageStyle Boolean false Adds Plumage style to the toast icon for Plumage styled toasts.
id String The id of the toast. This is used to identify the toast when calling the showToast method.
isStatus Boolean false If true, this sets the role of the toast from "alert" to "status" which is an ARIA role that is used to communicate the status of a process to screen readers. It also sets the aria-live attribute from "assertive" to "polite".
message String The message to be displayed in the toast. This is the main content of the toast.
noAnimation Boolean false If true, this will disable the animation of the toast when it is shown or hidden.
noCloseButton Boolean false If true, this will remove the close button from the toast.
noHoverPause Boolean false If true, this will prevent the toast from pausing the duration when the mouse hovers over it.
persistent Boolean false If true, this will make the toast persistent and it will not automatically hide after the duration has passed.
position String bottom-right The position of the toast. This can be set to "top-left", "top-center", "top-right", "bottom-left", "bottom-center", "bottom-right", "top-full", or "bottom-full".
plumageToast Boolean false If true, this will style the toast according to the Plumage style guide.
plumageToastMax Boolean false If true, this changes the styling of the Plumage toast so that more content can be displayed and adds the close button to the toast. To be used with the plumageToast prop.
solidToast Boolean false If true, this will style the toast as a solid minimal toast.
svgIcon String This uses predefined SVG icons included in the component, "check-circle-fill", "check-circle-outline", "info-fill", "info-outlined", "exclamation-triangle-fill", "exclamation-triangle-outline", "exclamation-circle-fill" or "exclamation-circle-outline".
time String The current ZULU formatted time to be displayed in the toast. This is used for Plumage styled toasts.
title String The title of the toast. This is displayed in the header of the toast.
variant String The variant color of the toast. This can be set to "primary", "secondary", "danger", "warning", "info", "dark" or "light". For Plumage styled toasts, this is applied to the icon.

Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

Install the necessary dependencies:

        
          npm install @vue/web-component-wrapper

Register the Lit Element Toast Component:

Create a file registerToastComponent.js:

          
            import Vue from 'vue';
            import wrap from '@vue/web-component-wrapper';
            import ToastComponent from './path/to/toast-component.js';

            const CustomElement = wrap(Vue, ToastComponent);
            window.customElements.define('toast-component', CustomElement);

Import and use in your Vue component:

In your main entry file (e.g., main.js):

            
              import Vue from 'vue';
              import App from './App.vue';
              import './registerToastComponent';

              new Vue({
                render: (h) => h(App),
              }).$mount('#app');

Using in a Vue component:

          
            <template>
            <div>
            <button @click="showToast">Show Toast</button>
            <toast-component id="toast" position="bottom-right"></toast-component>
            </div>
            </template>
            
            <script>
            export default {
              methods: {
                showToast() {
                  const toast = document.getElementById('toast');
                  toast.message = "This is a toast message!";
                  toast.variant = "primary";
                  toast.showToast();
                },
              },
            };
            </script>
React

Install the necessary dependencies:

        
          // bash
          npm install @lit-labs/react

Register the custom element:

Create a file registerToastComponent.js:

          
            import { createComponent } from '@lit-labs/react';
            import * as React from 'react';
            import { ToastComponent } from './path/to/toast-component.js';

            export const ToastComponentReact = createComponent({
              tagName: 'toast-component',
              elementClass: ToastComponent,
              react: React,
              events: {},
            });

Import and use in your React component:

          
            import React from 'react';
            import { ToastComponentReact } from './registerToastComponent';

            function App() {
              const showToast = () => {
                const toast = document.querySelector('toast-component');
                toast.message = "This is a toast message!";
                toast.variant = "primary";
                toast.showToast();
              };

              return (
                <div>
                <button onClick={showToast}>Show Toast</button>
                <ToastComponentReact id="toast" position="bottom-right" />
                </div>
              );
            }

            export default App;
Angular

Install the necessary dependencies:

        
          ng add @angular/elements

Register the custom element:

Create a file registerToastComponent.ts:

        
          import { Injector } from '@angular/core';
          import { createCustomElement } from '@angular/elements';
          import { ToastComponent } from './path/to/toast-component.js';

          export function registerCustomElements(injector: Injector) {
            const ToastElement = createCustomElement(ToastComponent, { injector });
            customElements.define('toast-component', ToastElement);
          }

Import and use in your Angular module:

In your app.module.ts:

          
            import { NgModule, Injector } from '@angular/core';
            import { BrowserModule } from '@angular/platform-browser';
            import { AppComponent } from './app.component';
            import { registerCustomElements } from './registerToastComponent';

            @NgModule({
              declarations: [AppComponent],
              imports: [BrowserModule],
              providers: [],
              bootstrap: [AppComponent],
            })
            export class AppModule {
              constructor(private injector: Injector) {
                registerCustomElements(injector);
              }

              ngDoBootstrap() {}
            }

Using in an Angular component:

        
          // html
          <!-- app.component.html -->
          <div>
          <button (click)="showToast()">Show Toast</button>
          <toast-component id="toast" position="bottom-right"></toast-component>
          </div>
        
      
          
            // app.component.ts
            import { Component } from '@angular/core';

            @Component({
              selector: 'app-root',
              templateUrl: './app.component.html',
            })
            export class AppComponent {
              showToast() {
                const toast = document.getElementById('toast') as any;
                toast.message = "This is a toast message!";
                toast.variant = "primary";
                toast.showToast();
              }
            }

Adjust the paths and imports based on your project structure. The provided examples showcase how to create wrapper components for each framework to use your Lit Element 'Table' component.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
