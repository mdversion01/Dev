Props
Property Type Default Description
checked Boolean false If true, the toggle switch is in the on position. If false, it is off.
disabled Boolean false If true, the switch is disabled. If false, it is enabled.
inline String This property is used for grouping multiple toggle switches into an inline layout. By default, a group of switches is stacked.
inputId String The id of the toggle switch. This is used to associate the label with the input.
labelTxt String The text of the label that is displayed next to the toggle switch.
newToggleTxt String An object with two properties, 'on' and 'off', that define the text that is displayed when the switch is toggled on and off.
noLabel Boolean false If true, the label is not displayed. If false, the label is displayed.
size String The size of the toggle switch. Possible values are "md" for medium and "lg" for large, when not used the size is default (small).

Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

To use the Lit Element component in Vue, you can use the 'vue-custom-element' library to register the Lit Element as a custom element:

This example assumes you have the ToggleSwitch component globally registered and available in your project. 

        
          // main.js
          import { ToggleSwitch } from './path/to/your/PlToggleSwitch.js';

          // Ensure the custom element is defined
          customElements.define('pl-toggle-switch', PlToggleSwitch);

          // Then import Vue and create your app
          import Vue from 'vue';
          import App from './App.vue';

          new Vue({
            render: h => h(App),
          }).$mount('#app');

Next, within your Vue component (App.vue in this example), you can use the pl-toggle-switch element directly in the template. You can bind properties and listen to events on it just like you would with a native HTML element or Vue component. 

          
            // javascript
            <template>
              <div>
               <h1>Toggle Switch Example</h1>
                <pl-toggle-switch
                  inputId="toggle-1"
                  labelTxt="Enable Feature"
                  :checked="isChecked"
                  @checked-changed="handleCheckChange"
                ></pl-toggle-switch>
            
                <!-- Example of using it within a loop -->
                <div v-for="(toggle, index) in toggles" :key="index">
                <pl-toggle-switch
                    :inputId="`toggle-${index}`"
                    :labelTxt="toggle.label"
                    :checked="toggle.checked"
                    @checked-changed="event => handleToggleChange(index, event)"
                    ></pl-toggle-switch>
                  </div>
                  </div>
                  </template>
            
                  <script>
                    export default {
                      data() {
                        return {
                          isChecked: false,
                          toggles: [
                            { label: 'Toggle 1', checked: false },
                            { label: 'Toggle 2', checked: true },
                          ],
                        };
                      },
                      methods: {
                        handleCheckChange(event) {
                          this.isChecked = event.detail.checked;
                        },
                        handleToggleChange(index, event) {
                          this.toggles[index].checked = event.detail.checked;
                        },
                      },
                    };
                  </script>
React

For React, you'll use refs to interact with the DOM element for properties and events not directly supported by React's synthetic event system: 

          
            // jsx
            import React, { useEffect, useRef } from 'react';
            import 'path/to/your/pl-toggle-switch-definition';
            
            const App = () => {
              const toggleRef = useRef(null);
            
              useEffect(() => {
                if (toggleRef.current) {
                  toggleRef.current.addEventListener('checked-changed', (e) => {
                    console.log(e.detail.checked);
                  });
                }
              }, []);
            
              return (
                <pl-toggle-switch
                  ref={toggleRef}
                  inputId="reactSwitch"
                  checked={true}
                  labelTxt="React Switch"
                ></pl-toggle-switch>
              );
            };
            
            export default App;
Angular

Angular can also work directly with Web Components. Use Angular's event binding to listen for custom events and property binding for attributes. First, ensure your custom element is declared to Angular's custom elements schema in your module: 

          
            // typescript 
            
            import { CUSTOM_ELEMENTS_SCHEMA, NgModule } from '@angular/core';
            import { BrowserModule } from '@angular/platform-browser';
            import { AppComponent } from './app.component';
            // Import your toggle switch somewhere suitable
            import 'path/to/your/pl-toggle-switch-definition';

            @NgModule({
              declarations: [AppComponent],
              imports: [BrowserModule],
              providers: [],
              bootstrap: [AppComponent],
              schemas: [CUSTOM_ELEMENTS_SCHEMA] // This line is important for custom elements
            })
            export class AppModule {}

Then, use it in your Angular component template:

          
            // html 

            <pl-toggle-switch
              inputId="angularSwitch"
              [attr.checked]="true"
              labelTxt="Angular Switch"
              (checked-changed)="handleSwitchChange($event)"
            ></pl-toggle-switch>

And in your component class:

          
            // typescript

            import { Component } from '@angular/core';

            @Component({
              selector: 'app-root',
              templateUrl: './app.component.html',
              styleUrls: ['./app.component.css']
            })
            export class AppComponent {
              handleSwitchChange(event: CustomEvent) {
                console.log(event.detail.checked);
              }
            }

Remember to adjust the paths and imports based on your project structure. These examples demonstrate how to create wrapper components for each framework to use your Lit Element 'CheckboxRadioInput' component.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
