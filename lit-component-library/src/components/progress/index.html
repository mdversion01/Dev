Property Type Default Description
animated Boolean false Adds an animation to the striped linear progress bar. The animation is left to right.
bars Array Used to add multiple progress bars to the progress component. Each item in the array should have the following properties: value, variant, striped, animated, showValue, showProgress, progressAlign, precision.
circular Boolean false Adds a circular progress bar to the component. The default is a linear progress bar.
height String 20 Sets the height of the linear progress bar. The default height is 20px.
indeterminate Boolean false Puts the circular progress bar into an indeterminate state to portray loading.
lineCap Boolean false Changes the styling of the ends of the circular progress bar from round to flat.
max Number 100 Sets the maximum value of the progress bar. The default is 100.
multi Boolean false Adds the ability to include multiple progress bars in the progress component.
precision Number 0 Sets the number of digits after the decimal for the progress bar value. The default is 0.
progressAlign String Aligns the text to the 'left' or 'right' of the value or progress for the linear progress bar. The default is center.
rotate Number 0 Sets the starting point of the circular progress bar. The default is 0 degrees.
showProgress Boolean false Adds a label to the progress bar that shows the percentage of the max value.
showValue Boolean false Adds a label to the progress bar that shows the current absolute value.
size Number 100 Sets the width and height of the circular progress bar. The default is 100px.
striped Boolean false Adds a stripe via CSS gradient over the linear progress barâ€™s background color.
styles String Adds custom inline styles to the progress component. The default is an empty string.
value Number 0 Sets the current value of the progress bar. The default is 0.
variant String primary This prop sets the color of the progress bar. The default in 'primary'. Other color options are secondary, success, danger, info, warning, and dark.

Using Component in other JavaScript Frameworks

Integrating a Lit Element component like Progress into frameworks such as Vue, React, and Angular involves wrapping the Web Component within the specific framework's component system. Here are brief guides on how to do this for each framework:

Vue.js

To use the Lit Element component in Vue, you can use the 'vue-custom-element' library to register the Lit Element as a custom element:

This example assumes you have the Modal component globally registered and available in your project. 

        
          // main.js
          import Vue from 'vue';
          import './path/to/your/progress'; // Ensure the Progress Web Component is imported

          // Prevent Vue from warning about the unknown `progress-component` element
          Vue.config.ignoredElements = ['progress-component'];

Next, within your Vue component (App.vue in this example), you can use the modal element directly in the template. You can bind properties and listen to events on it just like you would with a native HTML element or Vue component. 

          
            <template>
              <div>
                <progress-component value="50" max="100"></progress-component>
              </div>
            </template>
            
            <script>
            export default {
              name: 'App',
              // Additional component logic can be placed here
            }
            </script>
React

For React, you'll use refs to interact with the DOM element for properties and events not directly supported by React's synthetic event system: 

          
            // jsx
            import React from 'react';
            import './path/to/progress'; // Imports the Progress Web Component
            
            function App() {
              return (
                <div>
                <progress-component value={30} max={100}> </progress-component>
                </div>
              );
            }
            
            export default App;
Angular

Angular can also work directly with Web Components. Include Custom Elements Schema: Allow Angular to recognize custom elements. 

          
            // Add the CUSTOM_ELEMENTS_SCHEMA to your Angular module to avoid template parse errors.
            import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';
            import { BrowserModule } from '@angular/platform-browser';
            import { AppComponent } from './app.component';
            import './path/to/progress'; // Imports the Progress Web Component

            @NgModule({
              declarations: [
                AppComponent
              ],
              imports: [
                BrowserModule
              ],
              providers: [],
              bootstrap: [AppComponent],
              schemas: [CUSTOM_ELEMENTS_SCHEMA] // Allows custom elements
            })
            export class AppModule { }

Then, use it in your Angular component template:

          
            // html 
            
            <!-- In your component's HTML -->
            <progress-component [attr.value]="50" [attr.max]="100"></progress-component<

Remember to adjust the paths and imports based on your project structure. These examples demonstrate how to create wrapper components for each framework to use your Lit Element 'Modal' component.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.

Notes:

  • Event Handling: Make sure to handle events appropriately. React, for example, may require you to attach event listeners directly to the DOM elements for custom events, as shown above.
  • Properties and Attributes: If your Web Component uses properties that are objects or arrays, passing them from frameworks might require setting them via JavaScript rather than as HTML attributes.
