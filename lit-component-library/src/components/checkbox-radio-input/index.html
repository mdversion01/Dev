Props
Property Type Default Description
checked Boolean If true, the checkbox or radio button is checked. If false, it is unchecked.
checkbox Boolean If true, the component is a checkbox, if not, nothing is displayed or passed in as a prop.
customCheckbox Boolean If true, the component is a custom checkbox, if not, nothing is displayed or passed in as a prop.
customRadio Boolean If true, the component is a custom radio, if not, nothing is displayed or passed in as a prop.
customRadioRadio Boolean If true, the component is a custom radio group, if not, nothing is displayed or passed in as a prop.
disabled Boolean If true, the checkbox or radio button is disabled. If false, it is enabled.
formLayout String Used in the form component to set the layout of the checkbox or radio button. Vertical is the default layout if not used, and inline is the other option.
inputId String The id of the checkbox or radio button. This is used to associate the label with the input.
labelTxt String The text of the label that is displayed next to the checkbox or radio button.
name String The name of the radio group. This is used to group radio buttons together so that only one can be selected at a time.
noLabel Boolean If true, the label is not displayed. If false, the label is displayed.
radio Boolean If true, the component is a radio button, if not, nothing is displayed or passed in as a prop.
radioGroup Boolean If true, the component is a radio group, if not, nothing is displayed or passed in as a prop.
radioGroupOptions Array An array of objects that define the options for the radio group. Each object should have the following properties:
  • value: The value of the radio button
  • labelTxt: The text of the label that is displayed next to the radio button
  • inputId: The id of the radio button. This is used to associate the label with the input
  • disabled: If true, the radio button is disabled. If false, it is enabled
  • checked: If true, the radio button is checked. If false, it is unchecked
size String The size of the checkbox or radio button. Possible values are "sm" for small and "md" for medium, when not used the size is default.
selectedValue String The value of the selected radio button in the radio group.
validation Boolean If true, the component is validated. If false, it is not validated.
validationMessage String The validation message that is displayed when the component is invalid.
value String The value of the checkbox or radio button. This is the value that is sent to the server when the form is submitted.

Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

To use the Lit Element component in Vue, you can use the 'vue-custom-element' library to register the Lit Element as a custom element:

Install the vue-custom-element library:

        
          // bash
          npm install vue-custom-element

In your Vue component file, use the following code and in your Vue template, use the 'checkbox-radio-input-vue' custom element: 

          
            // javascript

            <template>
              <div>
                <!-- Other Vue components or HTML -->
                <checkbox-radio-input-vue 
                  :checked="true" 
                  :label-txt="'Checkbox Label'"
                ></checkbox-radio-input-vue>

                <!-- Other Vue components or HTML -->
              </div>
            </template>
            
            <script>
              import Vue from 'vue';
              import vueCustomElement from 'vue-custom-element';
              import 'path/to/checkbox-radio-input-component'; // Adjust the path
              
              Vue.use(vueCustomElement);
              
              Vue.customElement('checkbox-radio-input-vue', {
                props: ['checked', 'checkbox', 'customCheckbox', 'customRadio', 'customRadioGroup', 'disabled', 'inputId', 'noLabel', 'labelTxt', 'name', 'radio', 'radioGroup', 'radioGroupOptions', 'size', 'selectedValue', 'validation', 'validationMessage', 'value', 'formLayout'],
                template: '<checkbox-radio-input-component v-bind="$props"></checkbox-radio-input-component>',
              });
            </script>
            
            <style>
            /* Vue-specific styles */
            </style>
React

To use the Lit Element component in React, you can create a wrapper component and use the 'react-custom-element' library:

Install the react-custom-element library:

        
          // bash
          npm install react-custom-element

Create a wrapper component:

          
            // jsx

            import React from 'react';
            import { reactify } from 'react-custom-element';
            import 'path/to/checkbox-radio-input-component'; // Adjust the path

            const CheckboxRadioInputReact = reactify('checkbox-radio-input-react');

            const CheckboxRadioInputWrapper = (props) => (
              <CheckboxRadioInputReact {...props} />
            );

            export default CheckboxRadioInputWrapper;

In your React component, use the 'CheckboxRadioInputWrapper' component: 

          
            // jsx

            import React from 'react';
            import CheckboxRadioInputWrapper from './CheckboxRadioInputWrapper'; // Adjust the path

            const YourReactComponent = () => {
              return (
                <div>
                  <CheckboxRadioInputWrapper checked={true} labelTxt="Checkbox Label" />
                </div>
              );
            };

            export default YourReactComponent;
Angular

To use the Lit Element component in Angular, you can create a wrapper component and use Angular's 'createCustomElement' function:

Install the @angular/elements library:

        
          // bash
          npm install @angular/elements

Create an Angular wrapper component:

          
            // typescript 
            
            import { Component, Input, OnInit, OnDestroy, ElementRef, NgZone } from '@angular/core';
            import { createCustomElement } from '@angular/elements';
            
            declare var ng: any;
            
            @Component({
              selector: 'app-checkbox-radio-input-angular',
              template: '<checkbox-radio-input-component [checked]="checked" [label-txt]="labelTxt"></checkbox-radio-input-component>',
            })
            export class CheckboxRadioInputAngularComponent implements OnInit, OnDestroy {
              @Input() checked: boolean;
              @Input() labelTxt: string;
            
              constructor(private el: ElementRef, private ngZone: NgZone) {}
            
              ngOnInit() {
                const element = createCustomElement(CheckboxRadioInputAngularComponent, { injector: this.el.injector });
                customElements.define('checkbox-radio-input-angular', element);
                ng['zone'].run(() => {
                  customElements.define('checkbox-radio-input-angular', element);
                });
              }
            
              ngOnDestroy() {
                customElements.get('checkbox-radio-input-angular')?.remove();
              }
            }

In your Angular module, declare the wrapper component:

          
            // typescript 

            import { NgModule } from '@angular/core';
            import { BrowserModule } from '@angular/platform-browser';
            import { FormsModule } from '@angular/forms';
            import { CheckboxRadioInputAngularComponent } from './checkbox-radio-input-angular.component'; // Adjust the path

            @NgModule({
              declarations: [CheckboxRadioInputAngularComponent],
              imports: [BrowserModule, FormsModule],
              entryComponents: [CheckboxRadioInputAngularComponent],
            })
            export class AppModule {
              ngDoBootstrap() {}
            }

Remember to adjust the paths and imports based on your project structure. These examples demonstrate how to create wrapper components for each framework to use your Lit Element 'CheckboxRadioInput' component.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
