Props
Property Type Default Description
checked Boolean If true, the checkbox or radio button is checked. If false, it is unchecked.
checkbox Boolean If true, the component is a checkbox, if not, nothing is displayed or passed in as a prop.
disabled Boolean If true, the checkbox or radio button is disabled. If false, it is enabled.
formLayout String Used in the form component to set the layout of the checkbox or radio button. Vertical is the default layout if not used, and inline is the other option.
inputId String The id of the checkbox or radio button. This is used to associate the label with the input.
labelTxt String The text of the label that is displayed next to the checkbox or radio button.
name String The name of the radio group. This is used to group radio buttons together so that only one can be selected at a time.
noLabel Boolean If true, the label is not displayed. If false, the label is displayed.
radio Boolean If true, the component is a radio button, if not, nothing is displayed or passed in as a prop.
radioGroup Boolean If true, the component is a radio group, if not, nothing is displayed or passed in as a prop.
radioGroupOptions Array An array of objects that define the options for the radio group. Each object should have the following properties:
  • value: The value of the radio button
  • labelTxt: The text of the label that is displayed next to the radio button
  • inputId: The id of the radio button. This is used to associate the label with the input
  • disabled: If true, the radio button is disabled. If false, it is enabled
  • checked: If true, the radio button is checked. If false, it is unchecked
size String The size of the checkbox or radio button. Possible values are "sm" for small and "md" for medium, when not used the size is default.
selectedValue String The value of the selected radio button in the radio group.
validation Boolean If true, the component is validated. If false, it is not validated.
validationMessage String The validation message that is displayed when the component is invalid.
value String The value of the checkbox or radio button. This is the value that is sent to the server when the form is submitted.

Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

In a Vue component, you'd use the component just like any other HTML element: 

          
            <template>
            <div ref="container"></div>
            </template>

            <script>
            // Import your Lit Element Checkbox and Radio component
            import 'path-to-your-checkbox-radio-input-component'; // Update the path

            export default {
              props: ['inputId', 'labelTxt', 'checkbox', 'radio'],
              mounted() {
                const element = document.createElement('checkbox-radio-input-component');
                element.inputId = this.inputId;
                element.labelTxt = this.labelTxt;
                element.checkbox = this.checkbox;
                element.radio = this.radio;

                this.$refs.container.appendChild(element);
              },
            };
            </script>

            <style scoped>
            /* Add any scoped styles if needed */
            </style>
React

In a React component, you'd similarly treat the component as a regular HTML element: 

          
            // LitElementWrapper.jsx
            import React, { useRef, useEffect } from 'react';
            
            // Import your Lit Element Checkbox and Radio component
            import 'path-to-your-checkbox-radio-input-component'; // Update the path
            
            const LitElementWrapper = ({ inputId, labelTxt, checkbox, radio }) => {
              const containerRef = useRef();
            
              useEffect(() => {
                const element = document.createElement('checkbox-radio-input-component');
                element.inputId = inputId;
                element.labelTxt = labelTxt;
                element.checkbox = checkbox;
                element.radio = radio;
            
                containerRef.current.appendChild(element);
              }, [inputId, labelTxt, checkbox, radio]);
            
              return <div ref={containerRef}></div>;
            };
            
            export default LitElementWrapper;
Angular

In Angular, you'd use this component as part of your Angular template:

          
            // LitElementWrapper.component.ts
            import { Component, ElementRef, Input, OnDestroy, OnInit, Renderer2, ViewChild } from '@angular/core';

            // Import your Lit Element Checkbox and Radio component
            import 'path-to-your-checkbox-radio-input-component'; // Update the path

            declare global {
              interface Window {
                LitElement: any; // Adjust the type based on the LitElement class
              }
            }

            @Component({
              selector: 'app-lit-element-wrapper',
              template: '<div #container></div>',
            })
            export class LitElementWrapperComponent implements OnInit, OnDestroy {
              @Input() inputId: string;
              @Input() labelTxt: string;
              @Input() checkbox: boolean;
              @Input() radio: boolean;

              @ViewChild('container', { static: true }) containerRef: ElementRef;

              private litElement: any; // Adjust the type based on the LitElement class

              constructor(private renderer: Renderer2) {}

              ngOnInit(): void {
                this.litElement = document.createElement('checkbox-radio-input-component');
                this.litElement.inputId = this.inputId;
                this.litElement.labelTxt = this.labelTxt;
                this.litElement.checkbox = this.checkbox;
                this.litElement.radio = this.radio;

                this.renderer.appendChild(this.containerRef.nativeElement, this.litElement);
              }

              ngOnDestroy(): void {
                // Clean up, remove event listeners, etc.
              }
            }

These examples assume that you've configured your framework project to work with web components and that you've imported the Lit Element component correctly. Adjust import paths accordingly.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
