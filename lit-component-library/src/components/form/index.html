Props
Property Type Default Description
action String This property is used to specify the URL where to send the form data when the form is submitted. If this property is not set, the form will be sent to the current page.
fieldset Boolean When true, the form will be wrapped in a fieldset element. This property is used to group related elements in a form.
formLayout String horizontal, inline When use in the form component, it passes the layout setting from the parent to child component(s) within the <form> tag. The default is vertical.
formId String This property is used to specify the id of the form. It is used to link the form with an input.
legend Boolean false When true, this property is used to provide a caption for the fieldset element.
legendPosition String center, right This property is used to specify the position of the legend element. The default is left.
legendTxt String This property is used to specify the text of the legend element.
method String This property is used to specify the HTTP method to use when sending the form data. The default is get.
outsideOfForm Boolean false When true, this property is used to specify that the input element is outside of the form. Uses the formId property to link the input to the form.
bcolor String This property is used to specify the color of the border. The default is #CCCCCC. It is added as an inline style.
bradius Number This property is used to specify the radius of the border. Use a number to set, increments are defaulted to pixels. The default is 5. It is added as an inline style.
bstyle String This property is used to specify the style of the border. The default is solid. It is added as an inline style.
bwidth Number This property is used to specify the width of the border. The default is 1. It is added as an inline style.
styles String This property is used to specify the styles of the form. It is added as an inline style.
Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

In a Vue component, you'd use Form just like any other HTML element: 

          
            <template>
              <div>
              <!-- Include the Lit Element Form component as a Vue component -->
              <lit-element-form
                    :action="yourVueDataProperty"
                    :fieldset="yourVueDataProperty"
                    :form-layout="yourVueDataProperty"
                    :form-id="yourVueDataProperty"
                    :legend="yourVueDataProperty"
                    :legend-position="yourVueDataProperty"
                    :legend-txt="yourVueDataProperty"
                    :method="yourVueDataProperty"
                    :outside-of-form="yourVueDataProperty"
                    :bcolor="yourVueDataProperty"
                    :bradius="yourVueDataProperty"
                    :bstyle="yourVueDataProperty"
                    :bwidth="yourVueDataProperty"
                    :styles="yourVueDataProperty"
                  >
                  <!-- Your Vue.js content goes here -->
                  </lit-element-form>
                  </div>
                  </template>
              
                  <script>
              // Import the Lit Element Form component
              import 'lit-element-form'; // Replace with the correct path to your Lit Element Form component
              
              export default {
                data() {
                  return {
                    // Define your Vue.js data properties
                    yourVueDataProperty: 'value',
                  };
                },
                // Other Vue.js component options and lifecycle hooks go here
              };
              </script>
React

In a React component, you'd similarly treat Form as a regular HTML element: 

          
            import React from 'react';
            import 'lit-element-form'; // Replace with the correct path to your Lit Element Form component

            function YourReactComponent() {
              return (
                <div >
                  {/* Include the Lit Element Form component as a React component */}
                  <lit-element-form
                    action={yourReactDataProperty}
                    fieldset={yourReactDataProperty}
                    formLayout={yourReactDataProperty}
                    formId={yourReactDataProperty}
                    legend={yourReactDataProperty}
                    legendPosition={yourReactDataProperty}
                    legendTxt={yourReactDataProperty}
                    method={yourReactDataProperty}
                    outsideOfForm={yourReactDataProperty}
                    bcolor={yourReactDataProperty}
                    bradius={yourReactDataProperty}
                    bstyle={yourReactDataProperty}
                    bwidth={yourReactDataProperty}
                    styles={yourReactDataProperty}
                  >
                    {/* Your React content goes here */}
                    </lit-element-form >
                    </div>
              );
            }

            export default YourReactComponent;
Angular

In Angular, you'd use Form as part of your Angular template:

          
            import { Component } from '@angular/core';
            import 'path-to-lit-element-form'; // Replace with the correct path
            
            @Component({
              selector: 'app-your-angular-component',
              template: `
              <div>
              <!-- Use the LitElementForm custom element in your Angular template -->
              <lit-element-form
                    [action]="yourAngularDataProperty"
                    [fieldset]="yourAngularDataProperty"
                    <!-- Other bindings go here -->
                  >
                  <!-- Your Angular content goes here -->
                  </lit-element-form>
                  </div>
              `,
            })
            export class YourAngularComponent {
              // Define your Angular component logic here
            }

These examples assume that you've configured your framework project to work with web components and that you've imported the Lit Element component correctly. Adjust import paths accordingly.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
