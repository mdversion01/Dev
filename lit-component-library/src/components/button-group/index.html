This component is a wrapper for grouping buttons from the button component.

Props

Property Type Default Description
ariaLabel String Changes the default 'aria-label' of 'Button Group' to the inputed value.
Ex: ariaLabel="This is a button group"
vertical Boolean false Changes the default horizontal layout of the button group to a vertical layout.
Ex: vertical

Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

In a Vue component, you'd use ButtonGroup just like any other HTML element: 

        
          <template>
              <div>
                  <button-group :aria-label="ariaLabel" :class-names="classNames">
                      <pl-button slot="buttons">Button 1</pl-button>
                      <pl-button slot="buttons">Button 2</pl-button>
                      <pl-button slot="buttons">Button 3</pl-button>
                  </button-group>
              </div>
          </template>
          
          <script>
          import { defineComponent } from 'vue';
          
          export default defineComponent({
            data() {
              return {
                ariaLabel: 'Vue Button Group',
                classNames: 'my-custom-class',
                vertical: false,
              };
            },
          });
          </script>
React

In a React component, you'd similarly treat ButtonGroup as a regular HTML element: 

        
          import React from 'react';
          import ButtonGroup from 'path-to-button-group'; // Update the import path

          function MyReactComponent() {
            const ariaLabel = 'React Button Group';
            const classNames = 'my-custom-class';
            const vertical = false;

            return (
              <div>
                  <ButtonGroup ariaLabel={ariaLabel} classNames={classNames}>
                      <pl-button slot="buttons">Button 1</pl-button>
                      <pl-button slot="buttons">Button 2</pl-button>
                      <pl-button slot="buttons">Button 3</pl-button>
                  </ButtonGroup>
              </div>
            );
          }
Angular

In Angular, you'd use ButtonGroup as part of your Angular template:

        
          import { Component } from '@angular/core';

          @Component({
            selector: 'app-root',
            template: `
            <div>
                <button-group [ariaLabel]="'Angular Button Group'" [classNames]="'my-custom-class'">
                    <pl-button slot="buttons">Button 1</pl-button>
                    <pl-button slot="buttons">Button 2</pl-button>
                    <pl-button slot="buttons">Button 3</pl-button>
                </button-group>
            </div>
            `,
          })
          export class AppComponent {}

These examples assume that you've configured your framework project to work with web components and that you've imported the Lit Element component correctly. Adjust import paths accordingly.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
