Props
This component is a wrapper for grouping buttons from the button component.
Property Type Default Description
ariaLabel String Changes the default 'aria-label' of 'Button Group' to the input value.
Ex: ariaLabel="This is a button group"
vertical Boolean false Changes the default horizontal layout of the button group to a vertical layout.
Ex: vertical
Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

Assuming you have Vue components for PlButton and PlButtonGroup. 

          
            <template>
              <div>
              <!-- ButtonGroupVue.vue -->
               <pl-button-group :vertical="false" aria-label="Vue Button Group">
               <pl-button label="Button 1"></pl-button>
               <pl-button label="Button 2"></pl-button>
               <pl-button label="Button 3"></pl-button>
               </pl-button-group>
               </div>
               </template>

               <script>
            import PlButtonGroup from 'path-to-vue-pl-button-group'; // Adjust the import path
            import PlButton from 'path-to-vue-pl-button'; // Adjust the import path

            export default {
              components: {
                PlButtonGroup,
                PlButton,
              },
            };
            </script>
React

Assuming you have React components for PlButton and PlButtonGroup. 

          
            // ReactButtonGroup.jsx
            import React from 'react';
            import PlButtonGroup from 'path-to-react-pl-button-group'; // Adjust the import path
            import PlButton from 'path-to-react-pl-button'; // Adjust the import path

            const ReactButtonGroup = () => {
              return (
                <div>
                  <PlButtonGroup vertical={false} ariaLabel="React Button Group">
                    <PlButton label="Button 1" />
                    <PlButton label="Button 2" />
                    <PlButton label="Button 3" />
                  </PlButtonGroup>
                </div>
              );
            };

            export default ReactButtonGroup;
Angular

To use the Lit Element component in Angular, you can create a wrapper component and use Angular's 'createCustomElement' function:

Install the @angular/elements library:

        
          // bash
          npm install @angular/elements

Create an Angular wrapper component:

Assuming you have Angular components for PlButton and PlButtonGroup.

          
            // button-group-angular.component.ts
            import { Component } from '@angular/core';
            import { PlButtonGroupComponent } from 'path-to-angular-pl-button-group'; // Adjust the import path
            import { PlButtonComponent } from 'path-to-angular-pl-button'; // Adjust the import path

            @Component({
              selector: 'app-button-group-angular',
              template: `
              <div>
                <app-pl-button-group [vertical]="false" [ariaLabel]="'Angular Button Group'">
                  <app-pl-button label="Button 1"></app-pl-button>
                  <app-pl-button label="Button 2"></app-pl-button>
                  <app-pl-button label="Button 3"></app-pl-button>
                </app-pl-button-group>
              </div>
              `,
            })
            export class ButtonGroupAngularComponent {}

Make sure to adjust the paths and imports based on your project structure. These examples demonstrate how to create wrapper components for each framework to use your Lit Element ButtonGroup component.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
