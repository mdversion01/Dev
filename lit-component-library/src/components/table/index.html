Props
Styling
Property Type Default Description
bordered Boolean false Adds a border around the table. If set to true, the table will have a border around it.
borderless Boolean false Removes the borders from table. If set to true, the table will not have any borders.
dark Boolean false Adds a dark theme to the table. If set to true, the table will have a dark theme.
headerDark Boolean false Adds a dark theme to the table header. If set to true, the table header will have a dark theme.
headerLight Boolean false Adds a light theme to the table header. If set to true, the table header will have a light theme.
hover Boolean false Adds a hover effect to the table rows. If set to true, the table rows will have a hover effect.
plumage Boolean false Adds the Plumage styling to the table. If set to true, the table will have the Plumage styling.
noBorderCollapsed Boolean false Removes the border-collapse from the table. If set to true, the table will not have any border-collapse. This is useful when using sticky headers.
selectedVariant String table-active Adds a selected variant to the table rows. If set, the table rows will have a selected variant. This is useful when using selectable rows and you want to change the selected row color to a new class.
size String sm, lg Changes the size of the table and the other components within the table component, such as, pagination, input and select fields. If set to sm, the component will have a small size. If set to lg, the component will have a large size.
striped Boolean false Adds a striped effect to the table rows. If set to true, the table rows will have a striped effect.
tableVariant String table Adds a variant to the table. If set, the table will have a variant. Color options include: primary, secondary, success, danger, warning, info, light, and dark.
Layout
Property Type Default Description
caption String This adds and sets the placement of the table caption. It can be either set to 'top' or 'bottom' and the caption text can be added within the slot.
cloneFooter Boolean false Clones the header row and places it at the bottom of the table. If set to true, the header row will be cloned and placed at the bottom of the table.
fixed Boolean false This fixes all the table columns to the same width. If set to true, all the table columns will have the same width.
responsive Boolean false Makes the table responsive. If set to true, the table will be responsive.
stacked Boolean false Stacks the table columns on top of each other. If set to true, the table columns will be stacked on top of each other.
sticky Boolean false Makes the table header sticky. If set to true, the table header will be sticky when scrolling the table.
Data
Property Type Default Description
items Array [] The data to be displayed in the table. The data should be an array of objects.
Sorting and Filtering
Property Type Default Description
dropdownId String Used to connect the table to a dropdown component. The dropdown component should have the same id as the dropdownId.
filterText String The text to be used for filtering the table. The text should be a string.
sortCriteria Array [] The criteria to be used for sorting the table. The criteria should be an array of objects.
sortField String The field to be used for sorting the table. The field should be a string.
sortable Boolean false The table to be sorted. If set to true, the table will be sorted.
sortOrder String The order to be used for sorting the table. The order should be a string.
sortOrderDisabled Boolean false The order to be used for sorting the table. The order should be a string.
Selecting rows
Property Type Default Description
selectedFilterFields Array [] The fields to be used for filtering the table. The fields should be an array of objects.
selectedRows Array [] The rows to be selected in the table. The rows should be an array of objects.
selectMode String "single", "multi", "range" The mode to be used for selecting the rows in the table. The mode should be a string. Single allows only one row to be selected at a time. Multi allows multiple rows to be selected at a time. Range allows a range of rows to be selected at a time.
tableId String The id to be used for the table. The id should be a string. This is used for connecting the table to other components.
Table Pagination
Property Type Default Description
currentPage Number 1 This sets the current page of the table. The page should be a number.
goToButtons String By default, the 'Go to First', 'Previous', 'Next' and 'Go to Last' pagination buttons are displayed as icons. If set to "text", the pagination buttons will be displayed as text.
hideEllipsis Boolean false Hides the ellipsis in the pagination. If set to true, the ellipsis will be hidden.
hideGotoEndButtons Boolean false Hides the 'Go to First', 'Previous', 'Next' and 'Go to Last' pagination buttons when true. Only the page numbers will be displayed.
paginationLayout String The layout to be used for the pagination. The layout should be a string. Options include: for pagination layout 'center', 'end', fill (default is start) OR when used with showSizeChanger 'start', 'center', 'end', 'fill', 'fill-left', 'fill-right'.
paginationLimit Number 5 The number of pages to be displayed in the pagination at one time. The limit should be a number.
paginationPosition String The placement to be used for the pagination. The placement should be a string. Options include: 'top', 'bottom', 'both'.
paginationVariantColor String The color variant to be used for the pagination. The color variant should be a string. The default is primary with other options: secondary, tertiary, success, danger, warning, and info.
pageSizeOptions Array [] The options to be used for the 'Items per page' selection. This is used when showSizeChanger is set to true. The options should be an array of numbers and the default array is '[10, 20, 50, 100, "All"]'.
rowsPerPage Number 10 The number of rows to be displayed per page. The rows should be a number and the default is 10.
showDisplayRange Boolean false Shows the display range of the pagination, ex: '1-10 of 22'. If set to true, the display range will be shown.
showSizeChanger Boolean false Shows the 'Items per page' selection. If set to true, the 'Items per page' selection will be shown.
totalRows Number 0 The total number of rows in the table. The rows should be a number.
useByPagePagination Boolean false If set to true, the user can navigate through the pages using the 'Go to First', 'Previous', 'Next' and 'Go to Last' pagination buttons and an input filed to change the page number.
useMinimizePagination Boolean false If set to true, the pagination will be minimized and only show the 'Go to First', 'Previous', 'Next' and 'Go to Last' buttons.
usePagination Boolean false If set to true, the pagination will be displayed. If set to false, the pagination will not be displayed.
Slots
Slot Description
caption This is the slot for adding a caption for the table.
Add slot="caption" to a 'div' or 'span' in the table component.
Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

Install the necessary dependencies: 

        
          npm install lit
          npm install @webcomponents/webcomponentsjs --save

Register the Lit Element Table Component:

Create a TableComponent.vue file: 

          
            <template>
              <div>
                <table-component id="tableComponent" @row-selected="handleRowSelected"></table-component>
              </div>
            </template>
            
              <script>
            import 'path-to-lit-element-table-component.js';
            
            export default {
              name: 'TableComponent',
              methods: {
                handleRowSelected(event) {
                  console.log('Selected Rows:', event.detail);
                },
              },
              mounted() {
                const tableComponent = this.$el.querySelector('#tableComponent');
                tableComponent.items = [
                  { name: "John", age: 30, city: "New York" },
                  { name: "Anna", age: 25, city: "London" },
                  // more items...
                ];
                tableComponent.fields = [
                  { key: "name", label: "Name" },
                  { key: "age", label: "Age" },
                  { key: "city", label: "City" },
                ];
              }
            };
            </script>

Include the component in your Vue app:

          
            <template>
              <div id="app">
                <TableComponent />
              </div>
            </template>
            
            <script>
              import TableComponent from './components/TableComponent.vue';
              
              export default {
                name: 'App',
                components: {
                  TableComponent
                }
              };
            </script>
React

Install the necessary dependencies:

        
          // bash
          npm install lit
          npm install @webcomponents/webcomponentsjs --save

Create a React component for the Lit Element Table:

Create a TableComponent.jsx file:

          
            // TableComponent.jsx
            import React, { useEffect, useRef } from 'react';
            import 'path-to-lit-element-table-component.js';

            const TableComponent = () => {
              const tableRef = useRef(null);

              useEffect(() => {
                const tableComponent = tableRef.current;
                tableComponent.items = [
                  { name: "John", age: 30, city: "New York" },
                  { name: "Anna", age: 25, city: "London" },
                  // more items...
                ];
                tableComponent.fields = [
                  { key: "name", label: "Name" },
                  { key: "age", label: "Age" },
                  { key: "city", label: "City" },
                ];
                tableComponent.addEventListener('row-selected', handleRowSelected);
                return () => {
                  tableComponent.removeEventListener('row-selected', handleRowSelected);
                };
              }, []);

              const handleRowSelected = (event) => {
                console.log('Selected Rows:', event.detail);
              };

              return <table-component ref={tableRef}></table-component>;
            };

            export default TableComponent;

Include the component in your React app:

          
            import React from 'react';
            import ReactDOM from 'react-dom';
            import TableComponent from './components/TableComponent';

            const App = () => (
              <div>
                <TableComponent />
              </div>
            );

            ReactDOM.render(, document.getElementById('root'));
Angular

Install the necessary dependencies:

        
          npm install lit
          npm install @webcomponents/webcomponentsjs --save

Include the Web Components polyfills in polyfills.ts:

        
          import '@webcomponents/webcomponentsjs/webcomponents-bundle.js';

Create a custom Angular component to wrap the Lit Element Table:

Create a table-component-wrapper.component.ts file:

          
            import { Component, ElementRef, OnInit } from '@angular/core';
            import 'path-to-lit-element-table-component.js';

            @Component({
              selector: 'app-table-component-wrapper',
              template: '<table-component id="tableComponent"></table-component>',
              styles: []
            })
            export class TableComponentWrapper implements OnInit {
              constructor(private el: ElementRef) {}

              ngOnInit(): void {
                const tableComponent = this.el.nativeElement.querySelector('#tableComponent');
                tableComponent.items = [
                  { name: "John", age: 30, city: "New York" },
                  { name: "Anna", age: 25, city: "London" },
                  // more items...
                ];
                tableComponent.fields = [
                  { key: "name", label: "Name" },
                  { key: "age", label: "Age" },
                  { key: "city", label: "City" },
                ];
                tableComponent.addEventListener('row-selected', this.handleRowSelected);
              }

              handleRowSelected(event) {
                console.log('Selected Rows:', event.detail);
              }
            }

Include the component in your Angular module:

          
            import { BrowserModule } from '@angular/platform-browser';
            import { NgModule } from '@angular/core';
            import { AppComponent } from './app.component';
            import { TableComponentWrapper } from './components/table-component-wrapper/table-component-wrapper.component';

            @NgModule({
              declarations: [
                AppComponent,
                TableComponentWrapper
              ],
              imports: [
                BrowserModule
              ],
              providers: [],
              bootstrap: [AppComponent]
            })
            export class AppModule { }

Use the component in your Angular app:

          
            <app-table-component-wrapper></app-table-component-wrapper>

Adjust the paths and imports based on your project structure. The provided examples showcase how to create wrapper components for each framework to use your Lit Element 'Table' component.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
