Props
Property Type Default Description
absolute Boolean false Applies the inline style of position: absolute to the component.
ariaLabel String Changes the default 'aria-label' of 'Button' to the inputed value.
Ex: ariaLabel="This is a button"
block Boolean false Expands the button to 100% of available space.
bottom String number | string Aligns the component towards the bottom. This should be used with the absolute or fixed props.
Ex: bottom="50"
btnIcon Boolean false This property allows the user to make an icon into a button.
disabled Boolean false This property adds disabled to the button element.
end Boolean false Used with the button-group component. This property will add the 'pl-btn-group-end' or 'pl-btn-group-vertical-end' (when the property 'vertical' is used) class to the button.
fixed Boolean false If this property is added, an inline style with 'position: fixed' will be added to the component and it's position is will be fixed.
groupBtn Boolean false Designates the button as part of a 'Button Group' when used with the 'button-group' component.
iconBtn Boolean false Designates the button as a button that only uses an icon.
left String number | string Aligns the component towards the left. This should be used with the absolute or fixed props.
Ex: left="50"
link Boolean false Designates the button as an 'href' and not a 'button'.
outlined Boolean false When added, this makes the background transparent and applies a thin border.
pressed String Adding aria-pressed to an element with a role of button turns the button into a toggle button. The aria-pressed attribute is only relevant for toggle buttons. It represents the button's current "pressed" state. The value is "tri-state", meaning the value can be set to true, false, mixed, or undefined values. In the case of aria-pressed, like most tri-state value types, the default value is undefined. Toggle buttons require a full press-and-release cycle to change their value. Pressing and releasing it once changes the value to true. If it's pressed and released again, the value changes back to false. A value of mixed means that the values of more than one item controlled by the button do not all share the same value.
Ex: <pl-button variant="primary" pressed=" true | false | mixed ">Button Name</pl-button> will render aria-pressed="true | false | mixed"
right String number | string Aligns the component towards the right. This should be used with the absolute or fixed props.
Ex: right="50"
ripple Boolean false When added, a ripple effect is added to the button to indicate when the button is clicked.
shape String This property allows the user to change the shape of the button to square, pill, rounded(default) or circle (icon buttons).
Ex: shape="square"
size String This property allows the user to change the size of the button to xs(x-small), sm(small), default and lg(large).
Ex: size="sm"
start Boolean false Used with the button-group component. This property will add the 'pl-btn-group-start' or 'pl-btn-group-vertical-start' (when the property 'vertical' is used) class to the button.
styles String When using this property, additional inline styles can be added to the button.
Ex:<pl-button styles="color: pink; padding: 10px;"></pl-button>
text Boolean false Displays a button as just text with no background hover effects like with a or textBtn button.
textBtn Boolean false Displays a button as text but with the same hover effects as a button.
title String Add a title attribute to the button.
url String Used with the 'link' property, a url or path can be applied to the href that is rendered.
variant String This property allows the user to change the color of the button to one of the predefined colors/styles.
Ex: varian="primary"
vertical Boolean false Used with the button-group component. This property will add the 'pl-btn-group-vertical' class to the button, which will make the buttons stack vertically.
zIndex String number | string Specifies the stack order of an element. This should be used with the absolute or fixed props.
Ex: zIndex="50"
Styling

These class names can be added directly to the component similiar to Props.

Style Class Name Description
Color primary, secondary, tertiary, active-blue, success, danger, warning, light, dark These class names will set background color, border color and the hover colors for a button.
Ex: <pl-button variant="primary">Button Name</pl-button>
When using these class names with a 'text' prop, the class names will be appeneded with '--text' which will set background color and border color to transparent and set the color of the text and the hover colors.
Ex: <pl-button text variant="primary">Button Name</pl-button>
When using these class names with an 'outline' prop, the background color is transparent and it sets the border color, text color and the hover colors.
Ex: <pl-button variant="primary" outlined>Button Name</pl-button>
active and active-blue For active state, the class names will set the background color, border color and the hover colors.
When adding the 'active' class and the 'variant="active-blue"' class togather, the background color will be set to the active color and the hover color will be set to the active color.
Ex: <pl-button active variant="active-blue" >Button Name</pl-button>
Elevation flat, is-elevated, elevated-0 to elevated-24 The elevation helpers allow you to control relative depth, or distance, between two surfaces. There is a total of 25 elevation levels. You can set an elements elevation by using the class elevated-{n}, where n is a integer between 0-24 corresponding to the desired elevation. Applying flat will give give the button no depth and applying is-elevated will give the button a slight amount of depth..
Ex: <pl-button variant="primary" flat>Button Name</pl-button>
Ex: <pl-button variant="primary" is-elevated >Button Name</pl-button>
Ex: <pl-button variant="primary" elevated-3 >Button Name</pl-button>
Sizes xs, sm, default or add nothing, lg Sizing for text buttons. 'x-size="sm"' is 20px, 'size="sm"' is 28px, 'default' is 36px and 'large' is 44px.
Ex: <pl-button variant="primary" x-size="sm" >Button Name</pl-button>
Ex: <pl-button variant="primary" size="sm" >Button Name</pl-button>
Ex: <pl-button variant="primary" >Button Name</pl-button> (if no size is added, it defaults to default)
Ex: <pl-button variant="primary" size="lg" >Button Name</pl-button>
xs, sm, default or add nothing, lg Sizing for icon buttons. 'xs' is 1rem by 1rem, 'sm' is 28px by 28px, 'lg' is 44px by 44px and the default size, "default" or add nothing is 36px by 36px .
Ex: <pl-button variant="primary" icon-btn ></pl-button> (defsult size)
Ex: <pl-button variant="primary" icon-btn icon-xs ></pl-button>
Ex: <pl-button variant="primary" icon-btn icon-sm ></pl-button>
Ex: <pl-button variant="primary" icon-btn icon-lg ></pl-button>
Shape circle, pill, square By default, these buttons, whether buttons or icon buttons, the default will have rounded corners. circle' is to make the circular icon buttons. 'square' is to make the icon buttons, square or the buttons rectangle. 'pill' is to make the button a pill shape.
Ex: <pl-button variant="primary" shape="circle" iconBtn icon="fas fa-home" ></pl-button>
Ex: <pl-button variant="primary" shape="pill" >Button Name</pl-button>
Ex: <pl-button variant="primary" shape="square" >Button Name</pl-button>
PlButton Custom '_handleClick'

The _handleClick() method in the PlButton component is a custom event dispatcher that triggers the custom-click event when the button is clicked. To use this functionality in an instance of your PlButton component, a user would listen for the custom-click event.

Here's an example of how a user might implement this event listener: 

        
          // Creating an instance of the PlButton component
          const myButton = document.createElement('pl-button');

          // Attaching the component to the DOM
          document.body.appendChild(myButton);

          // Listening for the custom-click event
          myButton.addEventListener('custom-click', (event) => {
            console.log('Button was clicked!');
            // Perform actions or handle the event here
          });

By adding an event listener to the pl-button element, the user can respond to the custom-click event triggered by your component's _handleClick() method. This allows them to execute specific actions or code when the button is clicked.

To use the PlButton component in another JavaScript file, you'd typically import it and then use it within that file.

Here's an example of how you might do this:

          
            // Import the PlButton component
            import "./path/to/PlButton.js";

            // Use the PlButton in your code
            const button = document.createElement("pl-button");

            // Add properties, attributes, or event listeners if needed
            button.icon = "fas fa-home"; // Set an icon
            button.addEventListener("custom-click", () => {
              console.log("Button clicked!");
            });

            // Append the button to the document or any desired element
            document.body.appendChild(button);

This example demonstrates how you could import the PlButton component and then use it within your JavaScript file. You can set properties, attributes, or attach event listeners as needed, just like you would with any other HTML element.

Using Component in other JavaScript Frameworks
Vue.js

In a Vue component, you'd use PlButton just like any other HTML element: 

        
          <template>
          <div>
          <!-- Using the PlButton component -->
          <pl-button icon="fas fa-home" @custom-click="handleClick">Click me</pl-button>
          </div>
          </template>
          
          <script>
          // Import PlButton here if necessary
          export default {
            methods: {
              handleClick() {
                console.log('Button clicked!');
              }
            }
          };
          </script>
React

In a React component, you'd similarly treat PlButton as a HTML element: 

        
          import React from 'react';
          import './path/to/PlButton.js'; // Import PlButton here if necessary

          class MyComponent extends React.Component {
            handleClick() {
              console.log('Button clicked!');
            }

            render() {
              return (
                <div>
                  {/* Using the PlButton component */}
                  <pl-button icon="fas fa-home" onClick={this.handleClick}>Click me</pl-button>
                  </div>
              );
            }
          }
Angular

In Angular, you'd use PlButton as part of your Angular template:

        
          <!-- Inside your Angular component template -->
          <div>
          <!-- Using the PlButton component -->
          <pl-button [icon]="'fas fa-home'" (custom-click)="handleClick()">Click me</pl-button>
          </div>

        
      
        
          // Inside your Angular component class
          import { Component } from '@angular/core';

          @Component({
            selector: 'app-my-component',
            templateUrl: './my-component.component.html',
          })
          export class MyComponent {
            handleClick() {
              console.log('Button clicked!');
            }
          }

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
