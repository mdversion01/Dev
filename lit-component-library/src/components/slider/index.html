Props
Property Type Default Description
type String 'basic', 'multi', or 'discrete' The type of slider to render. 'basic' is a single slider, 'multi' is a range slider, and 'discrete' is a slider with discrete values.
disabled Boolean false Disables the slider so that it cannot be interacted with.
hideLeftTextBox Boolean false Hides the left side textbox that displays the current value. Used primarily with the Multi-Range Slider.
hideRightTextBox Boolean false Hides the right side textbox that displays the current value.
hideTextBoxes Boolean false Hides both the left and right side textboxes that display the current value. Used primarily with the Multi-Range Slider.
label String The label to display above the slider. If no label is provided, the slider will not display a label.
lowerValue Number The lower value of the range. Used primarily with the Multi-Range Slider.
max Number The maximum value of the slider. The default is 100. Used with 'basic' and 'multi' sliders.
min Number The minimum value of the slider. The default is 0. Used with 'basic' and 'multi' sliders.
plumage Boolean false Applies the Plumage styling to the slider thumb or thumbs.
plumage Boolean false Applies the Plumage styling to the slider thumb or thumbs.
selectedIndex Number The index of the selected value. Used primarily with the Discrete Slider.
sliderThumbLabel Boolean false Displays the current value of the slider on the slider thumb.
snapToTicks Boolean false Snaps the slider thumb to the nearest tick mark. Not needed if using the 'discrete' slider.
stringValues Array An array of strings to display as the tick labels. Used primarily with the Discrete Slider.
ticks Number The number of tick marks to display on the slider. Used primarily with the Basic and Multi-Range Sliders.
tickValues Array An array of numbers to display as the tick labels. Used primarily with the Basic and Multi-Range Sliders.
tickLabels Boolean false Displays the tick labels on the slider. Used primarily with the Basic and Multi-Range Sliders.
unit String The unit of measurement to display next to the current value.
upperValue Number The upper value of the range. Used primarily with the Multi-Range Slider.
variant String The color variant to apply to the slider. Options include 'primary', 'secondary', 'success', 'info', 'warning', 'danger', 'dark'.
value Number The current value of the slider. Used with 'basic' and 'multi' sliders.
Using Component in other JavaScript Frameworks

To integrate the slider-manager component and its three slider types (basic, multi, and discrete) into different JavaScript frameworks like Vue, React, and Angular, you need to provide guidelines on how to handle custom elements within these ecosystems.

Vue.js

Global Registration:
Register the custom elements globally in your main Vue.js entry file (e.g., main.js or app.js) to ensure they're recognized within the Vue templates. 

        
          // main.js
          import Vue from 'vue';
          import './path/to/slider-manager';
          import './path/to/basic-slider';
          import './path/to/multi-range-slider';
          import './path/to/discrete-slider';

          Vue.config.ignoredElements = [
            'slider-manager',
            'basic-slider',
            'multi-range-slider',
            'discrete-slider'
          ];

          new Vue({
            el: '#app'
          });

Once registered, you can use the sliders directly in your Vue templates. 

          
            <template>
              <div>
                <slider-manager type="basic" :value="50" min="0" max="100"></slider-manager>
              </div>
            </template>
React

You can create a React component wrapper for each custom element to handle properties and events effectively. 

          
            // jsx
            import React from 'react';

            class BasicSlider extends React.Component {
              componentDidMount() {
                this.node.addEventListener('value-changed', this.handleValueChange);
              }

              componentWillUnmount() {
                this.node.removeEventListener('value-changed', this.handleValueChange);
              }

              handleValueChange = (event) => {
                if (this.props.onValueChange) {
                  this.props.onValueChange(event.detail);
                }
              }

              render() {
                return (
                  <basic-slider ref={node => this.node = node} {...this.props}></basic-slider>
                );
              }
            }

            export default BasicSlider;

Usage in React Components:
Use the wrapper component as you would any other React component. 

            
              import React from 'react';
              import BasicSlider from './components/BasicSlider';

              function App() {
                return (
                  <div>
                  <BasicSlider value={50} min={0} max={100} onValueChange={(detail) => console.log(detail)} />
                      </div>
                );
              }

              export default App;
Angular

Module Configuration:
Configure your Angular module to include custom elements schema. 

          
            // typescript 
            import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';
            import { BrowserModule } from '@angular/platform-browser';
            import { AppComponent } from './app.component';

            @NgModule({
              declarations: [
                AppComponent
              ],
              imports: [
                BrowserModule
              ],
              providers: [],
              bootstrap: [AppComponent],
              schemas: [CUSTOM_ELEMENTS_SCHEMA]  // Important for custom elements
            })
            export class AppModule { }

Component Usage:
Use the custom elements directly in Angular templates. 

          
            // html 
            
            <slider-manager type="multi" [attr.value]="value" min="0" max="100"></slider-manager>

General Note for All Frameworks
Ensure that the web components are loaded and defined in the global scope before your framework-based application initializes. This ensures that the custom elements are parsed correctly by the browser's custom element registry. Each framework has its specifics for handling non-reactive attributes and events, so adjustments might be necessary based on more complex interactions or state management needs.

By following these guidelines, you can integrate Lit-based web components like slider-manager and its related sliders into Vue, React, and Angular applications effectively, leveraging the encapsulated functionality of web components across different frontend ecosystems.

Notes:

  • Event Handling: Make sure to handle events appropriately. React, for example, may require you to attach event listeners directly to the DOM elements for custom events, as shown above.
  • Properties and Attributes: If your Web Component uses properties that are objects or arrays, passing them from frameworks might require setting them via JavaScript rather than as HTML attributes.
