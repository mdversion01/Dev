Input Field Sizes
Props
Property Type Default Description
disabled Boolean false When this is true, the input field is disabled and users cannot interact with it.
formLayout String vertical, horizontal, inline This changes the layout of the input field. The default is vertical.
size String sm, lg This changes the size of the input field. The default is default.
label String This is the label for the input field. When used the label is added to the aria-labelledby, for, id, and placeholder (unless placeholder prop is used) attribute.
labelHidden Boolean false This hides the label to be used by screen readers and uses it as the placeholder text (unless placeholder prop is used).
required Boolean false This makes the input field required and displays an asterisk next to the label.
type String text, email, password, url, tel, search, hidden, date This is the type of input field. The default is text.
validation Boolean false When true, this will display the input in red.
validationMessage String This is the message that will be displayed when the input field is in a validation state.
value String This is the value of the input field.

Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

In a Vue component, you'd use InputField just like any other HTML element: 

          
            <template>
              <div>
              <!-- Other Vue components or HTML -->
                <input-field-component
                  :disabled="false"
                  :formLayout="'horizontal'"
                  :size="'lg'"
                  :label="'Your Label'"
                  :labelHidden="false"
                  :required="true"
                  :search="false"
                  :type="'text'"
                  :validation="false"
                  :validationMessage="'Your validation message'"
                  :value="'Your initial value'"
                  ></input-field-component>
                  <!-- Other Vue components or HTML -->
                </div>
              </template>
            
            <script>
            import 'lit-fontawesome';
            // Import the Lit Element component
            import './path-to-lit-element-component/input-field-component.js';
            
            export default {
              // Vue component options
            };
            </script>
            
            <style>
            /* Vue-specific styles */
            </style>
React

In a React component, you'd similarly treat InputField as a regular HTML element: 

          
            import React from 'react';
            import 'lit-fontawesome';
            // Import the Lit Element component
            import './path-to-lit-element-component/input-field-component.js';

            function YourReactComponent() {
              return (
                <div>
                  {/* Other React components or JSX */}
                  <input-field-component
                    disabled={false}
                    formLayout="horizontal"
                    size="lg"
                    label="Your Label"
                    labelHidden={false}
                    required={true}
                    search={false}
                    type="text"
                    validation={false}
                    validationMessage="Your validation message"
                    value="Your initial value"
                    ></input-field-component>
                  {/* Other React components or JSX */}
                  </div>
              );
            }

            export default YourReactComponent;
Angular

In Angular, you'd use InputField as part of your Angular template:

          
            import { Component } from '@angular/core';
            import 'lit-fontawesome';
            // Import the Lit Element component
            import './path-to-lit-element-component/input-field-component.js';

            @Component({
              selector: 'your-angular-component',
              template: `
              <div>
              <!-- Other Angular components or HTML -->
                  <input-field-component
                    [disabled]="false"
                    [formLayout]="'horizontal'"
                    [size]="'lg'"
                    [label]="'Your Label'"
                    [labelHidden]="false"
                    [required]="true"
                    [search]="false"
                    [type]="'text'"
                    [validation]="false"
                    [validationMessage]="'Your validation message'"
                    [value]="'Your initial value'"
                    ></input-field-component>
                    <!-- Other Angular components or HTML -->
                  </div>
              `,
              styles: [/* Angular-specific styles */]
            })
            export class YourAngularComponent {
              // Angular component logic
            }

These examples assume that you've configured your framework project to work with web components and that you've imported the Lit Element component correctly. Adjust import paths accordingly.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
