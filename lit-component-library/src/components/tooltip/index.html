Tooltip
Basic Usage

Basic tooltips are a good choice to provide the user with concise basic application and content information.

Users can set the placement of a tooltip by assigning 'top', 'bottom', 'left', 'right', and 'auto' to the 'position' or 'data-placement' props. When 'auto' is specified, it will dynamically reorient the tooltip.

The placement of a tooltip will flip if there is not enough space in the specified direction.

Contextual tooltips are meant to use color cuing to draw the user's eye to the tooltip and provide additional information at-a-glance.

Colors can be set using the 'variant' or 'data-variant' props. Preset colors are 'primary', 'secondary', 'danger', 'warning', 'info', 'success', and 'dark'. Custom classes can be made as well and assign to the 'customClass' prop.

Props
Property Type Default Description
animation Boolean false If set to true, the tooltip will fade in and out on hover.
container OR data-container Boolean false Appends the tooltip to a specific element. Example: container: 'body'. This option is particularly useful in that it allows you to position the tooltip in the flow of the document near the triggering element - which will prevent the tooltip from floating away from the triggering element during a window resize.
customClass String Add a custom class to the tooltip. Multiple classes can be added by separating them with a space.
htmlContent Boolean false Insert HTML into the tooltip. Use text if you're worried about XSS attacks.
message String The message to display in the tooltip if not using the 'data-original-title' or 'title' attribute.
position OR data-placement String top How to position the tooltip - top | bottom | left | right | auto. When 'auto' is specified, it will dynamically reorient the tooltip.
title OR data-original-title String Set title value if 'title' attribute isn't present.
trigger OR data-trigger String hover focus How tooltip is triggered - click | hover | focus | manual. You may pass multiple triggers; separate them with a space.
variant OR data-variant String Contextual color to use for the tooltip - primary | secondary | danger | warning | info | success | dark.
Using Component in other JavaScript Frameworks
Vue.js

Global Registration:
Register the custom elements globally in your main Vue.js entry file (e.g., main.js or app.js) to ensure they're recognized within the Vue templates.

Register the Custom Element in Vue:
Use Vue's config to ignore the custom element during compilation.

Use the Custom Element in a Vue Component:

         
           // main.js
           import Vue from 'vue';
            import App from './App.vue';
            import './tooltip-component.js'; // Path to your TooltipComponent

            Vue.config.ignoredElements = [/tooltip-component/];

            new Vue({
              render: h => h(App),
            }).$mount('#app');

Once registered, you can use the tooltip directly in your Vue templates. 

           
            
            <!-- App.vue -->
              <template>
              <div id="app">
              <tooltip-component 
                    id="tooltip1" 
                    data-placement="top" 
                    data-original-title="Initial tooltip content" 
                    data-trigger="hover focus" 
                    data-custom-class="custom-top-class" 
                    data-variant="success">
                    <button type="button" class="btn btn-secondary">Hover to see tooltip</button>
                    </tooltip-component>
                    </div>
                    </template>

                    <script>
                      export default {
                        methods: {
                          updateTooltipContent() {
                            const tooltipElement = document.getElementById('tooltip1');
                            const newContent = `Updated tooltip content: ${new Date().toLocaleTimeString()}`;
                            tooltipElement.setAttribute('data-original-title', newContent);

                            // Manually hide and show the tooltip to apply the new content
                            tooltipElement.hideTooltip();
                            tooltipElement.showTooltip();
                          }
                        }
                      }
                    </script>
React

Install and Import the Web Component:
Ensure your custom element is included in your project. You can include the custom element in your main entry point file.

Use the Custom Element in a React Component:

           
            // index.js
            import React from 'react';
            import ReactDOM from 'react-dom';
            import App from './App';
            import './tooltip-component.js'; // Path to your TooltipComponent
            
            ReactDOM.render(
              <React.StrictMode>
              <App />
              </React.StrictMode>,
              document.getElementById('root')
            );
           
         
             
              // App.js
              import React from 'react';
              
              class App extends React.Component {
                updateTooltipContent = () => {
                  const tooltipElement = document.getElementById('tooltip1');
                  const newContent = `Updated tooltip content: ${new Date().toLocaleTimeString()}`;
                  tooltipElement.setAttribute('data-original-title', newContent);
              
                  // Manually hide and show the tooltip to apply the new content
                  tooltipElement.hideTooltip();
                  tooltipElement.showTooltip();
                };
              
                render() {
                  return (
                    <div id="app">
                    <tooltip-component 
                        id="tooltip1" 
                        data-placement="top" 
                        data-original-title="Initial tooltip content" 
                        data-trigger="hover focus" 
                        data-custom-class="custom-top-class" 
                        data-variant="success">
                        <button type="button" className="btn btn-secondary">Hover to see tooltip</button>
                        </tooltip-component>
                        </div>
                  );
                }
              }
              
              export default App;
Angular

Install and Import the Web Component:
Ensure your custom element is included in your project. You can include the custom element in your main entry point file.

Include Custom Elements Schema in Angular Module:

           
             // typescript 
             // main.ts
              import './tooltip-component.js'; // Path to your TooltipComponent
              import { platformBrowserDynamic } from '@angular/platform-browser-dynamic';
              import { AppModule } from './app/app.module';

              platformBrowserDynamic().bootstrapModule(AppModule)
                .catch(err => console.error(err));
           
         
           
            // app.module.ts
            import { BrowserModule } from '@angular/platform-browser';
            import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';

            import { AppComponent } from './app.component';

            @NgModule({
              declarations: [
                AppComponent
              ],
              imports: [
                BrowserModule
              ],
              schemas: [CUSTOM_ELEMENTS_SCHEMA],
              providers: [],
              bootstrap: [AppComponent]
            })
            export class AppModule { }
            
         
          
            <!-- app.component.html -->
            <div id="app">
            <tooltip-component 
                id="tooltip1" 
                data-placement="top" 
                data-original-title="Initial tooltip content" 
                data-trigger="hover focus" 
                data-custom-class="custom-top-class" 
                data-variant="success">
                <button type="button" class="btn btn-secondary">Hover to see tooltip</button>
                </tooltip-component>
                </div>
           
        
          
            // app.component.ts
            import { Component } from '@angular/core';
            
            @Component({
              selector: 'app-root',
              templateUrl: './app.component.html',
              styleUrls: ['./app.component.css']
            })
            export class AppComponent {
              updateTooltipContent() {
                const tooltipElement = document.getElementById('tooltip1');
                const newContent = `Updated tooltip content: ${new Date().toLocaleTimeString()}`;
                tooltipElement.setAttribute('data-original-title', newContent);
            
                // Manually hide and show the tooltip to apply the new content
                tooltipElement.hideTooltip();
                tooltipElement.showTooltip();
              }
            }

This will set up the TooltipComponent for use in Vue, React, and Angular. The custom element is registered, and the styles are injected into the document head, ensuring that they are applied to the tooltips created outside of the shadow DOM.

General Note for All Frameworks
Ensure that the web components are loaded and defined in the global scope before your framework-based application initializes. This ensures that the custom elements are parsed correctly by the browser's custom element registry. Each framework has its specifics for handling non-reactive attributes and events, so adjustments might be necessary based on more complex interactions or state management needs.

By following these guidelines, you can integrate Lit-based web components like tooltip into Vue, React, and Angular applications effectively, leveraging the encapsulated functionality of web components across different frontend ecosystems.

Notes:

  • Event Handling: Make sure to handle events appropriately. React, for example, may require you to attach event listeners directly to the DOM elements for custom events, as shown above.
  • Properties and Attributes: If your Web Component uses properties that are objects or arrays, passing them from frameworks might require setting them via JavaScript rather than as HTML attributes.
