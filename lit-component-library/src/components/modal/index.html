Props

The modal launch button uses these props from the button component: 'block', 'classnames', 'disabled', 'id', 'outlined', 'ripple' and 'styles'. See the documentation here.

Property Type Default Description
btnText String The text displayed on the button that launches the modal. It is also used to set the aria-label attribute on the button.
modalId String The id of the modal. This is used to uniquely identify the modal component. It is also used to set the aria-labelledby attribute on the modal and the aria-controls attribute on the button that launches the modal as well as the data-target attribute on the button that closes the modal.
modalLg Boolean false Set to true to make the modal large. This will increase the width of the modal to 90% of the viewport width.
modalSm Boolean false Set to true to make the modal small. This will decrease the width of the modal to 50% of the viewport width.
modalTitle String The title of the modal. This is displayed at the top of the modal.
scrollableBody Boolean flase Set to true to make the body of the modal scrollable. This is useful when the content of the modal is too long to fit in the viewport.
scrollLongContent Boolean false Set to true to make the entire modal scrollable. This is useful when the content of the modal is too long to fit in the viewport.
verticallyCentered Boolean false Set to true to vertically center the modal.

Slot Description
footer Use this slot to add additional buttons/pl-button component or content to the footer of the modal. This is useful for adding buttons that perform actions like saving or deleting data.
PlButton Custom '_handleClick'

The _handleClick() method in the PlButton component is a custom event dispatcher that triggers the custom-click event when the button is clicked. To use this functionality in an instance of your PlButton component, a user would listen for the custom-click event.

Here's an example of how a user might implement this event listener in the Modal component: 

        
         // Dispatching the custom-click event
          const modal = document.getElementById("modal2");

          // Creating an instance of the PlButton component
          const saveButton = document.getElementById("saveButton");

          // Listening for the custom-click event
          // Assuming the modal component has a method closeModal() exposed 
          saveButton.addEventListener("click", () => {
            console.log("Save button clicked");
            // Code to save data goes here or for editing data or deleting data etc.
            // Close the modal after saving data
            modal.closeModal(); // Call closeModal method on the modal-component
          });

By adding an event listener to the pl-button element, the user can respond to the custom-click event triggered by your component's _handleClick() method. This allows them to execute specific actions or code when the button is clicked.

Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

To use the Lit Element component in Vue, you can use the 'vue-custom-element' library to register the Lit Element as a custom element:

This example assumes you have the Modal component globally registered and available in your project. 

        
          // main.js
          import Vue from 'vue';
          import App from './App.vue';
          
          // Import your custom element here
          import 'path/to/modal-component.js';
          
          new Vue({
            el: '#app',
            render: h => h(App)
          });

Next, within your Vue component (App.vue in this example), you can use the modal element directly in the template. You can bind properties and listen to events on it just like you would with a native HTML element or Vue component. 

          
            // javascript
            // Tell Vue to ignore the custom element
            Vue.config.ignoredElements = ['modal-component'];

            <template>
                <div>
                    <modal-component open @close-modal="handleClose">
                        <!-- Modal content here -->
                    </modal-component>
                </div>
            </template>
              
            <script>
              export default {
                methods: {
                  handleClose() {
                    console.log('Modal closed');
                  }
                }
              };
            </script>
React

For React, you'll use refs to interact with the DOM element for properties and events not directly supported by React's synthetic event system: 

          
            // jsx
            import React from 'react';

            class App extends React.Component {
              componentDidMount() {
                document.addEventListener('close-modal', this.handleClose);
              }
            
              componentWillUnmount() {
                document.removeEventListener('close-modal', this.handleClose);
              }
            
              handleClose = () => {
                console.log('Modal closed');
              }
            
              render() {
                return (
                    <modal-component open>
                    {/* Your modal content here */}
                    </modal-component>
                );
              }
            }
            
            export default App;
Angular

Angular can also work directly with Web Components. Use Angular's event binding to listen for custom events and property binding for attributes. First, ensure your custom element is declared to Angular's custom elements schema in your module: 

          
            // Add the CUSTOM_ELEMENTS_SCHEMA to your Angular module to avoid template parse errors.
            
            import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';
            import { BrowserModule } from '@angular/platform-browser';
            import { AppComponent } from './app.component';

            @NgModule({
                declarations: [
                    AppComponent
                ],
                imports: [
                    BrowserModule
                ],
                providers: [],
                bootstrap: [AppComponent],
                schemas: [CUSTOM_ELEMENTS_SCHEMA] // Add this line
            })
            export class AppModule { }

Then, use it in your Angular component template:

          
            // html 
            
            <!-- In your component template -->
            <modal-component [attr.open]="isOpen" (close-modal)="handleClose()">
                <!-- Your modal content here -->
            </modal-component>

And in your component class:

          
            // typescript

            // In your component TypeScript
            import { Component } from '@angular/core';
            
            @Component({
              selector: 'app-root',
              templateUrl: './app.component.html'
            })
            export class AppComponent {
              isOpen = true;
            
              handleClose() {
                console.log('Modal closed');
                this.isOpen = false;
              }
            }

Remember to adjust the paths and imports based on your project structure. These examples demonstrate how to create wrapper components for each framework to use your Lit Element 'Modal' component.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.

Notes:

  • Event Handling: Make sure to handle events appropriately. React, for example, may require you to attach event listeners directly to the DOM elements for custom events, as shown above.
  • Properties and Attributes: If your Web Component uses properties that are objects or arrays, passing them from frameworks might require setting them via JavaScript rather than as HTML attributes.
