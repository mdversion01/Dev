Props
Property Type Default Description
accordion Boolean false When this is true, the collapse component will look and behave like an accordion.
contentTxtSize String This property allows you to change the overall size of the text (xs, sm, default, lg, xl, or xxl) that is in the content area of the collapse component.
icon Array ["fas fa-angle-down"] This property uses the Font Awesome fa-angle-down(caret down) icon by default. It rotates the icon 180 degrees when the collapse component opens and again when it closes. You can pass in an array of two icons to change the icon when the collapse component is open and closed. The first icon in the array is the icon when the collapse component is closed, and the second icon is the icon when the collapse component is open.
Ex: icon="fas fa-minus-circle, fas fa-plus-circle"
isOpen Boolean false When true, the collapse component will be open by default.
targetId String Providing a targetId will allow the collapse component to toggle. The targetId is used by 'aria-controls', 'aria-labelledby', 'data-target' attributes of the button and 'id' of the content area.
toggle String To use when implementing and Accordion, when property is set to 'single', only one collapse component can be opened at a time.
Other The collapse component also accepts other properties from the Button component. Those properties are: block, variant, size, disabled, ripple, and link, and they can be found here - Button API
Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

In a Vue component, you'd use Collapse just like any other HTML element: 

          
            <template>
              <div>
                <collapse-component targetId="collapse-1" toggle="single" :accordion="true" contentTxtSize="sm">
                  <span slot="expansion-header">Default Toggle</span>
                  <div slot="content">This content can be collapsed</div>
                </collapse-component>
              </div>
            </template>
            
            <script>
            import 'path_to_your_libraries/collapse.js'; // Import your Collapse component
            
            export default {
              // Your Vue component definition
            };
            </script>
React

In a React component, you'd similarly treat Collapse as a regular HTML element: 

          
            import React from 'react';
            import 'path_to_your_libraries/collapse.js'; // Import your Collapse component

            function App() {
              return (
                <div>
                  <collapse-component targetId="collapse-1" toggle="single" accordion={true} contentTxtSize="sm">
                    <span slot="expansion-header">Default Toggle</span>
                    <div slot="content">This content can be collapsed</div>
                  </collapse-component>
                </div>
              );
            }

            export default App;
Angular

In Angular, you'd use Collapse as part of your Angular template:

          
            // Import the required decorators and other necessary modules

            @Component({
              selector: 'app-root',
              template: `
                <div>
                  <collapse-component targetId="collapse-1" [toggle]="'single'" [accordion]="true" [contentTxtSize]="'sm'">
                    <span slot="expansion-header">Default Toggle</span>
                    <div slot="content">This content can be collapsed</div>
                  </collapse-component>
                </div>
              `
            })
            export class AppComponent {
              // Your Angular component logic
            }

These examples assume that you've configured your framework project to work with web components and that you've imported the Lit Element component correctly. Adjust import paths accordingly.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
