Props
Property Type Default Description
custom Boolean false When this is true, the select field is styled to look like a custom select field.
defaultTxt String This is the default text that is displayed when the select field is empty.
disabled Boolean false When this is true, the select field is disabled and users cannot interact with it.
formLayout String horizontal, inline This changes the layout of the select field. The default is vertical. If select field component is used inside a form component, then formLayout prop of form component will be applied to select field component and therefore it does not have to be added to the select field component. Also, remember to import your components in the correct order.
formId String This is the id of the form. When used, the id is added to the form and to the select field's form attribute.
selectFieldId String This is the id of the select field. When used, the id is added to the select field and to the label's 'for' attribute.
options Array This is the options for the select field. The array should be in the format: [{value: 'value', name: 'name', selected: true}]
size String sm, lg This changes the size of the select field. The default is default.
label String This is the label for the select field. When used the label is added to the aria-label and placeholder (unless placeholder prop is used) attribute.
labelHidden Boolean false This hides the label to be used by screen readers and uses it as the placeholder text (unless placeholder prop is used).
multiple Boolean false When this is true, the select field allows multiple selections.
required Boolean false This makes the select field required and displays an asterisk next to the label.
validation Boolean false When true, this will display the select field in red.
validationMessage String This is the message that will be displayed when the select field is in a validation state.
value String This is the value of the select field.
Slots
Slot Description
formField This is the slot for form fields. It is used when using inside a form component.
Add slot="formField" to the input field component.
Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

Assuming you have Vue components for PlSelectField.

In your Vue component file, use the following code and in your Vue template, use the 'select-field-vue' custom element: 

          
            <template>
              <div>
                <!-- SelectFieldVue.vue -->
                <pl-select-field :label="'Vue Select Field'" :options="vueOptions"></pl-select-field>
              </div>
            </template>

              <script>
            import PlSelectField from 'path-to-vue-select-field'; // Adjust the import path

            export default {
              components: {
                PlSelectField,
              },
              data() {
                return {
                  vueOptions: [
                    { value: 'option1', name: 'Option 1' },
                    { value: 'option2', name: 'Option 2' },
                    { value: 'option3', name: 'Option 3' },
                  ],
                };
              },
            };
            </script>
React

Assuming you have React components for PlSelectField.

          
            // ReactSelectField.jsx
            import React from 'react';
            import PlSelectField from 'path-to-react-select-field'; // Adjust the import path

            const ReactSelectField = () => {
              const reactOptions = [
                { value: 'option1', name: 'Option 1' },
                { value: 'option2', name: 'Option 2' },
                { value: 'option3', name: 'Option 3' },
              ];

              return (
                <div>
                  <PlSelectField label="React Select Field" options={reactOptions} />
                </div>
              );
            };

            export default ReactSelectField;
Angular

Assuming you have Angular components for PlSelectField.

          
            // select-field-angular.component.ts
            import { Component } from '@angular/core';
            import { PlSelectFieldComponent } from 'path-to-angular-select-field'; // Adjust the import path
            
            @Component({
              selector: 'app-select-field-angular',
              template: `
                <div>
                  <app-pl-select-field label="Angular Select Field" [options]="angularOptions"></app-pl-select-field>
                </div>
              `,
            })
            export class SelectFieldAngularComponent {
              angularOptions = [
                { value: 'option1', name: 'Option 1' },
                { value: 'option2', name: 'Option 2' },
                { value: 'option3', name: 'Option 3' },
              ];
            }

Adjust the paths and imports based on your project structure. The provided examples showcase how to create wrapper components for each framework to use your Lit Element 'SelectField' component.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
