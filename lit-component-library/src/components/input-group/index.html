Input Group Sizing
Props
Property Type Default Description
append Boolean false When this is true, the input field will have an append icon or text.
appendId String This is the id of the append icon or text. This is used to associate the input field with the append icon or text.
disabled Boolean false When this is true, the input field is disabled and users cannot interact with it.
formLayout String horizontal, inline This changes the layout of the input group field. The default is vertical. If input group component is used inside a form component, then formLayout prop of form component will be applied to input group component and therefore it does not have to be added to the input group component. Also, remember to import your components in the correct order.
icon String This property is used to add an icon to the append or prepend portion of the input field. Currently only Font Awesome icons are supported.
inputId String This is the id of the input field. When used, the id is added to the input and to the label's 'for' attribute.
label String This is the label for the input field. When used the label is added to the aria-label and placeholder (unless placeholder prop is used) attribute.
labelHidden Boolean false This hides the label to be used by screen readers and uses it as the placeholder text (unless placeholder prop is used).
placeholder String This is the placeholder text for the input field. When used, the placeholder is added to the input field. If not used, the label will be used as the placeholder text.
prepend Boolean false When this is true, the input field will have a prepend icon or text.
prependId String This is the id of the prepend icon or text. This is used to associate the input field with the prepend icon or text.
required Boolean false This makes the input field required and displays an asterisk next to the label.
size String sm, lg This changes the size of the input field. The default is default.
type String text, email, password, url, tel, search, hidden, date This is the type of input field. The default is text.
validation Boolean false When true, this will display the input in red.
validationMessage String This is the message that will be displayed when the input field is in a validation state.
value String This is the value of the input field.
Slots
Slot Description
input This is the slot for the input field. It is used when using inside a form component.
Add slot="input" to the input field component.
Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

In a Vue component, you'd use InputGroup just like any other HTML element: 

          
            <template>
              <div>
              <!-- Other Vue components or HTML -->
              <input-group-component
                    :append="true"
                    :appendId="'append-icon-id'"
                    :disabled="false"
                    :formLayout="'horizontal'"
                    :icon="'fa fa-search'"
                    :inputId="'your-input-id'"
                    :inputSize="'lg'"
                    :label="'Your Label'"
                    :labelHidden="false"
                    :placeholder="'Your Placeholder'"
                    :prepend="true"
                    :prependId="'prepend-icon-id'"
                    :required="true"
                    :size="'md'"
                    :type="'text'"
                    :validation="false"
                    :validationMessage="'Your Validation Message'"
                    :value="'Your Initial Value'"
                    ></input-group-component>
                  <!-- Other Vue components or HTML -->
                  </div>
                  </template>
              
                  <script>
              // Import the Lit Element component
              import './path-to-lit-element-component/input-group.js';
              
              export default {
                // Vue component options
              };
              </script>
              
              <style>
              /* Vue-specific styles */
              </style>
React

In a React component, you'd similarly treat InputGroup as a regular HTML element: 

          
            import React from 'react';
            // Import the Lit Element component
            import './path-to-lit-element-component/input-group.js';
            
            function YourReactComponent() {
              return (
                <div>
                  {/* Other React components or JSX */}
                  <input-group-component
                    append={true}
                    appendId="append-icon-id"
                    disabled={false}
                    formLayout="horizontal"
                    icon="fa fa-search"
                    inputId="your-input-id"
                    inputSize="lg"
                    label="Your Label"
                    labelHidden={false}
                    placeholder="Your Placeholder"
                    prepend={true}
                    prependId="prepend-icon-id"
                    required={true}
                    size="md"
                    type="text"
                    validation={false}
                    validationMessage="Your Validation Message"
                    value="Your Initial Value"
                    ></input-group-component>
                  {/* Other React components or JSX */}
                  </div>
              );
            }
            
            export default YourReactComponent;
Angular

In Angular, you'd use InputGroup as part of your Angular template:

          
            import { Component } from '@angular/core';
            // Import the Lit Element component
            import './path-to-lit-element-component/input-group.js';
            
            @Component({
              selector: 'your-angular-component',
              template: `
              <div>
              <!-- Other Angular components or HTML -->
              <input-group-component
                    [append]="true"
                    [appendId]="'append-icon-id'"
                    [disabled]="false"
                    [formLayout]="'horizontal'"
                    [icon]="'fa fa-search'"
                    [inputId]="'your-input-id'"
                    [inputSize]="'lg'"
                    [label]="'Your Label'"
                    [labelHidden]="false"
                    [placeholder]="'Your Placeholder'"
                    [prepend]="true"
                    [prependId]="'prepend-icon-id'"
                    [required]="true"
                    [size]="'md'"
                    [type]="'text'"
                    [validation]="false"
                    [validationMessage]="'Your Validation Message'"
                    [value]="'Your Initial Value'"
                    > </input-group-component>
                  <!-- Other Angular components or HTML -->
                  </div>
              `,
              styles: [/* Angular-specific styles */]
            })
            export class YourAngularComponent {
              // Angular component logic
            }

These examples assume that you've configured your framework project to work with web components and that you've imported the Lit Element component correctly. Adjust import paths accordingly.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
