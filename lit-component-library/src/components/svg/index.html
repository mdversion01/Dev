SVG Component

The SVG component is used to display SVG icons. Just copy and paste SVG code into the component tag. The properties below can be used to style the SVG element.

Props
Property Type Default Description
arialHidden Boolean false The 'ariaHidden' property can be used when using an SVG as an icon. 'ariaHidden="true" is rendered as 'aria-hidden="true" in the rendered SVG.
ariaLabel String The 'ariaLabel' property will add 'ariaLabel="value" is rendered as 'aria-label="value"' in the rendered SVG.
fill String The 'fill' property can be used to change the color of the SVG. By default the 'fill="currentColor', in which the SVG will inherit the color of the parent element.
Ex: 'fill="#ffc0cb"' ( add a new color, rgb, rgba or hsl values of a color can be used as well) in the component will render to 'fill="#ffc0cb"' in the SVG.
height Number The 'height' property can be used to change the height of the SVG. Ex: height="48" in the component will render to the SVG.
svgMargin String Used in buttons with text and SVG's as icons, the 'svgMargin' property is used to wrap a span around the SVG add margin to an SVG when added to the left or right of text in a button. Ex: svgMargin="left" in the component will render to and wrap a <span class="mr-1"></span> around the SVG. svgMargin="right" will render "ml-1" each will give the SVG some space between it's self and the button text.
width Number The 'width' property can be used to change the width of the SVG. Ex: width="48" in the component will render to the SVG.

Ensure to handle the compatibility between custom elements and the Angular lifecycle for proper usage.

In Angular, you may need to add the custom element (`svg-component`) to the `schemas` array in your `AppModule` to make it available across your Angular app. The `schemas` array in `AppModule` allows Angular to understand custom elements.

Here's an example of how you might do this:

        
          // Import necessary modules
          import { BrowserModule } from '@angular/platform-browser';
          import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';

          import { AppComponent } from './app.component';

          @NgModule({
            declarations: [
              AppComponent
            ],
            imports: [
              BrowserModule
            ],
            schemas: [CUSTOM_ELEMENTS_SCHEMA], // Add this line to include custom elements
            providers: [],
            bootstrap: [AppComponent]
          })
          export class AppModule { }

By adding `CUSTOM_ELEMENTS_SCHEMA` to the `schemas` array, Angular will recognize the custom elements like `svg-component`. After adding this to your `AppModule`, you should be able to use the `svg-component` tag in your Angular templates without any issues.

Please note that in each example, you need to ensure that you've imported the `svg-component` from the correct file path or library location where the component is defined. Additionally, the attribute bindings might slightly vary depending on the framework (for example, in Angular, attribute binding is done using square brackets `[]`).
