Props
Property Type Default Description
arrowOff Boolean false Removes the arrow from the popover. This is useful when the popover is used as a dropdown.
content String The content to display in the popover. This can be HTML content.
customClass String Add a custom class to the tooltip. Multiple classes can be added by separating them with a space.
fallbackPlacement String flip Auto-flip placement behaviour of the popover, relative to the trigger element. flip(is the default), clockwise, counterclockwise, or an array of valid placements evaluated from left to right.
offset Number Offset of the popover relative to the trigger element. This can be used to adjust the position of the popover left or right.
placement String auto Position of the popover relative to the trigger element. Options are auto, top, bottom, left, right, topleft, topright, bottomleft, bottomright, lefttop, leftbottom, righttop, rightbottom.
plumage Boolean false Adds Plumage styling to the popover.
super Boolean false Adds super-tooltip styling to the popover. Used with the Plumage prop.
target Object Reference to the element that the popover should be attached to.
title OR data-original-title String Set title value if 'title' attribute isn't present.
trigger String click How tooltip is triggered - click | hover | focus | manual. You may pass multiple triggers; separate them with a space.
variant String Contextual color to use for the tooltip - primary | secondary | danger | warning | info | success | dark.
yoffset Number Offset of the popover relative to the trigger element. This can be used to adjust the position of the popover up or down.
Using Component in other JavaScript Frameworks
Vue.js

Global Registration:
Register the custom elements globally in your main Vue.js entry file (e.g., main.js or app.js) to ensure they're recognized within the Vue templates.

Register the Custom Element in Vue:
Use Vue's config to ignore the custom element during compilation.

Use the Custom Element in a Vue Component:

         
           // main.js
           import { createApp } from 'vue';
            import App from './App.vue';
            import './components/PopoverComponent.js'; // Adjust the path accordingly

            createApp(App).mount('#app');

Once registered, you can use the popover directly in your Vue templates. 

           
            <!-- App.vue -->
              <template>
              <div>
                    <popover-component 
                    title="Popover Title"
                    content="This is the popover content."
                    placement="top"
                    trigger="click">
                    <button>Click me</button>
                    </popover-component>
                  </div>
                </template>
              
              <script>
              export default {
                name: 'PopoverExample'
              }
              </script>
React

Install and Import the Web Component:
Ensure your custom element is included in your project. You can include the custom element in your main entry point file.

Register the Custom Element: In your main JavaScript file or component file, you need to register the custom element.

           
            import React from 'react';
            import ReactDOM from 'react-dom';
            import './components/PopoverComponent.js'; // Adjust the path accordingly
            import App from './App';

            ReactDOM.render( <App / >, document.getElementById('root'));

Use the Component in React: In your React component, use the custom element as you would any HTML element.

             
                import React from 'react';

                const PopoverExample = () => (
                    <div>
                    <popover-component 
                      title="Popover Title"
                      content="This is the popover content."
                      placement="top"
                      trigger="click">
                      <button>Click me</button>
                      </popover-component>
                      </div>
                );
                
                export default PopoverExample;
Angular

Install and Import the Web Component:
Ensure your custom element is included in your project. You can include the custom element in your main entry point file.

Register the Custom Element: In your main.ts file, you need to import the custom element.

           
             // typescript 
             // main.ts
             import { platformBrowserDynamic } from '@angular/platform-browser-dynamic';
             import { AppModule } from './app/app.module';
             import './components/PopoverComponent.js'; // Adjust the path accordingly
             
             platformBrowserDynamic().bootstrapModule(AppModule)
               .catch(err => console.error(err));

Use the Component in Angular: In your Angular component, use the custom element as you would any HTML element.

           
            // app.component.html
            <div>
            <popover-component 
                title="Popover Title"
                content="This is the popover content."
                placement="top"
                trigger="click">
                <button>Click me </button>
                </popover-component>
                </div>

This will set up the PopoverComponent for use in Vue, React, and Angular. The custom element is registered, and the styles are injected into the document head, ensuring that they are applied to the tooltips created outside of the shadow DOM.

General Note for All Frameworks
Ensure that the web components are loaded and defined in the global scope before your framework-based application initializes. This ensures that the custom elements are parsed correctly by the browser's custom element registry. Each framework has its specifics for handling non-reactive attributes and events, so adjustments might be necessary based on more complex interactions or state management needs.

By following these guidelines, you can integrate Lit-based web components like tooltip into Vue, React, and Angular applications effectively, leveraging the encapsulated functionality of web components across different frontend ecosystems.

Notes:

  • Event Handling: Make sure to handle events appropriately. React, for example, may require you to attach event listeners directly to the DOM elements for custom events, as shown above.
  • Properties and Attributes: If your Web Component uses properties that are objects or arrays, passing them from frameworks might require setting them via JavaScript rather than as HTML attributes.
