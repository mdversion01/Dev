Icon Component

The icon component is used to display icons. The icon component can be used with the font awesome icons.

Props
Property Type Default Description
arialHidden Boolean false By default, icons will have 'aria-hidden="true"' when rendered, if 'araiHidden' property is used, 'aria-hidden="false" is then rendered to the icon.
This currently only works with the icons and not the svg element.
ariaLabel String The ariaLabel property will add 'aria-label="value"' to the icon.
This currently only works with the icons and not the svg element.
color String Using the color property and a color, will add an inline style to the icon.
Ex: color="#ffc0cb" (the name of the color, rgb, rgba or hsl values of a color can be used as well) in the component will render to <i style="color: #ffc0cb;"></i>.
icon String The icon property is used to display an icon and setting the value to the icon's classes. Ex: icon="fas fa-home" in the component will render to <i class="fas fa-home"></i>.
iconPosition String The iconPosition property is used to add margin to an icon when added to the left or right of text in a button. Ex: icon="fas fa-home" iconPosition="left" in the component will render to <i class="fas fa-home mr-1"></i>. iconPosition="right" will render "ml-1"
iconSize String The iconSize property is used to add CSS class lg, sm, and xs sizes to the icons. The icons are a default size of 1rem when the iconSize property is not used. Ex: icon="fas fa-home" in the component will render to <i class="fas fa-home"></i>.
size Number Using the size property and giving it a numeric value, will add an inline style to the icon.
Ex: size="48" in the component will render to <i style="font-size: 48px;"></i>.
tokenIcon Boolean false This property is used to display an icon within a badge token. It icons size is inherit from the badge token.
Ex: <icon-component icon="fas fa-home" tokenIcon ></icon-component>.
Using Component in other JavaScript Frameworks
Vue.js

Here's how the Icon component would be used in a Vue.js component:

        
          <template>
          <div>
          <icon-component
                icon="fas fa-user"
                ariaLabel="User Icon"
                ariaHidden="true"
                iconPosition="left"
                iconSize="20px"
                color="blue"
                ></icon-component>
              <p>Vue Component with Lit Element Icon</p>
              </div>
              </template>
          
              <script>
          import 'icon-component'; // Import the Lit Element Icon component
          
          export default {
            // Vue component properties, methods, etc.
          };
          </script>
React

Here's how the Icon component would be used in a React component:

        
          import React from 'react';
          import 'icon-component'; // Import the Lit Element Icon component

          const MyReactComponent = () => {

              <icon-component

                  color="blue"
                  > </icon-component>
                <p>React Component with Lit Element Icon </p>
                </div>

          };

          export default MyReactComponent;
Angular

Here's how the Icon component would be used in an Angular component:

Basic usage:

            <icon-component

In Angular, using Lit Element components directly might require additional steps due to differences in the way Angular handles the DOM. You might need to create a wrapper or use a library like ngx-build-plus to enable Angular to handle custom elements.

The approach would involve creating a custom Angular component that wraps the Lit Element Icon component. 

        
          // IconWrapperComponent.ts
          import { Component, ElementRef, ViewChild, AfterViewInit } from '@angular/core';

          @Component({
            selector: 'app-icon-wrapper',
            template: '<div #iconContainer></div>'
          })
          export class IconWrapperComponent implements AfterViewInit {
            @ViewChild('iconContainer', { static: false }) iconContainer: ElementRef;

            ngAfterViewInit() {
              import('icon-component').then(() => {
                const iconElement = document.createElement('icon-component');
                this.iconContainer.nativeElement.appendChild(iconElement);
              });
            }
          }

Ensure to handle the compatibility between custom elements and the Angular lifecycle for proper usage.

In Angular, you may need to add the custom element (`icon-component`) to the `schemas` array in your `AppModule` to make it available across your Angular app. The `schemas` array in `AppModule` allows Angular to understand custom elements.

Here's an example of how you might do this:

          
            // Import necessary modules
            import { BrowserModule } from '@angular/platform-browser';
            import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';

            import { AppComponent } from './app.component';

            @NgModule({
              declarations: [
                AppComponent
              ],
              imports: [
                BrowserModule
              ],
              schemas: [CUSTOM_ELEMENTS_SCHEMA], // Add this line to include custom elements
              providers: [],
              bootstrap: [AppComponent]
            })
            export class AppModule { }

By adding `CUSTOM_ELEMENTS_SCHEMA` to the `schemas` array, Angular will recognize the custom elements like `icon-component`. After adding this to your `AppModule`, you should be able to use the `icon-component` tag in your Angular templates without any issues.

Please note that in each example, you need to ensure that you've imported the `icon-component` from the correct file path or library location where the component is defined. Additionally, the attribute bindings might slightly vary depending on the framework (for example, in Angular, attribute binding is done using square brackets `[]`).
