Props
Property Type Default Description
alignMenuRight Boolean false When true, aligns the dropdown menu to the right of the button
block Boolean false When true, the dropdown menu will take up the full width of its parent
buttonText String The text to display on the dropdown button. If not provided, the default text will be "Dropdown".
disabled Boolean false When true, the dropdown button will be disabled and the dropdown menu will not be displayed.
dotsIcon Boolean false When true, the button will display the dots icon only. The 'buttonText' prop provides the 'aria-label' text for accessibility. This is typically used for dropdown buttons that are part of a menu.
gearIcon Boolean false When true, the button will display the gear icon only. The 'buttonText' prop provides the 'aria-label' text for accessibility. This is typically used for dropdown buttons that are part of a menu.
iconDropDown Boolean false When true, the button will display an icon button only, the 'dotsIcon' or the 'gearIcon' props can be used. The 'buttonText' prop provides the 'aria-label' text for accessibility. This is typically used for dropdown buttons that are part of a menu.
id String The unique identifier for the dropdown component. If not provided, a unique id will be generated.
items Array An array of objects representing the items in the dropdown menu. Each object should have a 'name' property and an optional 'path' property. If the 'path' property is not provided, the item will not be clickable and will be rendered as a disabled item. If the 'isDivider' property is true, the item will be rendered as a divider. Adding a 'submenu' property to an item will render the item as a submenu item. The 'submenu' property should be an array of objects representing the submenu items.
outlined Boolean false When true, the button will be rendered with an outlined style.
pressed Boolean false When true, the button will be rendered with a pressed style.
ripple Boolean false When true, the button will be rendered with a ripple effect.
showDropdown Boolean false When true, the dropdown menu will be displayed. This prop is typically used to control the visibility of the dropdown menu from outside the component.
shape String The shape of the button. Possible values are 'default', 'square', and 'pill'.
size String The size of the button. Possible values are 'pl-size'(Plumage), 'sm', 'lg', and default is when there isn't a prop assigned.
styles String Custom styles to apply to the dropdown button. The value should be a string of CSS styles.
focusedIndex Number The index of the item in the dropdown menu that should receive focus when the menu is opened. This prop is typically used to control the focus of the dropdown menu from outside the component.
variant String The variant of the button. Possible values are 'primary', 'secondary', 'tertiary', 'active', 'success', 'danger', 'warning', 'info', 'light', and 'dark'.
Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Vue.js

Vue has good support for Web Components. In a Vue component, you'd use Dropdown just like any other HTML element: 

          
            <template>
            <div>
            <dropdown-component button-text="Click me!" id="example-dropdown"></dropdown-component>
            </div>
            </template>
          
            <script>
          import 'path/to/your/dropdown-component';
          
          export default {
            name: 'App'
          }
          </script>
React

React's JSX doesn't natively listen to changes in non-standard properties or events on a web component. For properties, you might need to use refs to interact with your component imperatively. In a React component, you'd similarly treat Dropdown as a regular HTML element: 

          
            import React, { useEffect, useRef } from 'react';
            import 'path/to/your/dropdown-component';
            
            function App() {
              const dropdownRef = useRef(null);
            
              useEffect(() => {
                if(dropdownRef.current) {
                  dropdownRef.current.buttonText = 'Click me!';
                  dropdownRef.current.id = 'example-dropdown';
                  // Add any event listeners if needed
                }
              }, []);
            
              return (
                <dropdown-component ref={dropdownRef}></dropdown-component>
              );
            }
            
            export default App;
Angular

Angular has built-in support for custom elements through Angular Elements. To use your Web Component, you might need to add it to the schemas array in your module to avoid Angular's template checks. In Angular, you'd use Dropdown as part of your Angular template: 

          
            import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';
            import { BrowserModule } from '@angular/platform-browser';
            import { AppComponent } from './app.component';

            // Import the custom element somewhere in your app
            import 'path/to/your/dropdown-component';

            @NgModule({
              declarations: [
                AppComponent
              ],
              imports: [
                BrowserModule
              ],
              providers: [],
              bootstrap: [AppComponent],
              schemas: [CUSTOM_ELEMENTS_SCHEMA] // Add this line
            })
            export class AppModule { }

Then you can use it directly in your Angular component's HTML:

        
          <dropdown-component buttonText="Click me!" id="example-dropdown"></dropdown-component>

These examples assume that you've configured your framework project to work with web components and that you've imported the Lit Element component correctly. Adjust import paths accordingly.

Remember, for Angular and React, to ensure that any custom events your component fires are properly handled, you might need to add event listeners manually using refs in React or ViewChild in Angular, since handling custom events directly in the template (React) or template (Angular) might not work as expected without some additional setup.

Note

Make sure the custom element (dropdown-component) definition is loaded before you try to use it in your app. This often means importing the component script in your main entry file or using a module bundler like Webpack to include it as part of your build process.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
