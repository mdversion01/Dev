Props
Property Type Default Description
append Boolean true This adds an append area to the input field. For these components it contains the calendar icon and for the Date Picker and the Date Range Picker, they are buttons. The default is true.
appendId String This is the id of the append area. When used, the id is added to the append area.
ariaLabel String This is the aria-label for the div that the displays the calendar or calendars. When used, the aria-label is added to the div.
calendar Boolean false When added to the Date Picker component, this will display the stand alone date picker calendar. The default is false. Used only with Date Picker component.
dateFormat String YYYY-MM-DD This is the format of the date. The default is YYYY-MM-DD. It can be set to MM-DD-YYYY instead of the default. The format is based on the moment.js library.
disabled Boolean false When this is true, the input field is disabled and users cannot interact with it.
displayContextExamples Boolean false This is used to display the context examples for the Date Picker component. The default is false.
formLayout String horizontal, inline This changes the layout of the input field. The default is vertical. If input component is used inside a form component, then formLayout prop of form component will be applied to input component and therefore it does not have to be added to the input component. Also, remember to import your components in the correct order.
inputId String This is the id of the input field. When used, the id is added to the input and to the label's 'for' attribute.
is24HourFormat Boolean true This is used with the Date Range Time Picker component. When true, the time is displayed in 24 hour format, when false the time is displayed in 12 hour format. The default is true.
joinBy String - This is the text that is displayed between the start and end dates in the Date Range and Date Range Time Picker components. The default is a hyphen, '-' but you can change the prop to 'joinBy="to"', which is the other acceptable format . This is used only with the Date Range and Date Range Time Picker component.
label String This is the label for the input field. When used the label is added to the aria-label and placeholder (unless placeholder prop is used) attribute.
labelHidden Boolean false This hides the label to be used by screen readers and uses it as the placeholder text (unless placeholder prop is used).
placeholder String This is the placeholder text for the input field. When used, the placeholder is added to the input field. By default, this uses what has been set in the dateFormat prop.
plumage Boolean false This changes the Plumage styles theme. The default is false.
prepend Boolean false This adds a prepend area to the input field. For these components it contains the calendar icon and for the Date Picker and the Date Range Picker, they are buttons. The default is false.
prependId String This is the id of the prepend area. When used, the id is added to the prepend area.
rangePicker Boolean false When added to the Date Range Picker component, this will display the stand alone date range picker calendars. The default is false. Used only with Date Range Picker component.
rangeTimePicker Boolean false When added to the Date Range Time Picker component, this will display the stand alone date range and time picker calendars. The default is false. Used only with Date Range Time Picker component.
required Boolean false This makes the input field required and displays an asterisk next to the label.
showDuration Boolean false This is used with the Date Range Time Picker component. When true, the duration of the time is displayed. The default is false.
size String sm, lg This changes the size of the input field. The default is default.
validation Boolean false When true, this will display the input in red.
validationMessage String This is the message that will be displayed when the input field is in a validation state.
value String This is the value of the input field.
warningMessage String This is the message that will be displayed when the input field is in a warning state.
Using Component in other JavaScript Frameworks

Integrating a Lit Element component with other JavaScript frameworks like Vue, React, and Angular involves creating a wrapper component in each framework to encapsulate and use the Lit Element component. Here are examples for each framework:

Dependencies

Hereâ€™s a breakdown of the main dependencies you'll likely need for these components:

1. Lit (Core library for web components):

  • Since the components are written in Lit, you need to install lit in each project (Vue, React, Angular) to support the custom elements. 
          
            // bash
            npm install lit

2. Popper.js (Positioning of dropdowns/popups):

  • Popper.js is used to position elements like date and time dropdowns. 
          
            // bash
            npm install @popperjs/core

Font Awesome (Icons):

  • Font Awesome is used for icons (e.g., calendar icon).
          
            // bash
            npm install @fortawesome/fontawesome-free

Web Component Support:

  • Ensure your framework supports custom elements. For Vue, React, and Angular, there are specific configurations to allow the use of custom elements like the date picker components.
Vue.js

To make sure Vue supports custom elements, update vue.config.js or add the following to the main Vue file: 

        
          Vue.config.ignoredElements = [/date-picker/, /date-range-picker/, /date-range-time-picker/];

Step 1: Installation

Install the required packages as before.

Step 2: Register Custom Elements

In main.js, register the Lit components as custom elements:

          
             // main.js
             import Vue from 'vue';
             import vueCustomElement from 'vue-custom-element';
             import 'path-to-datepicker-package/datepicker';
             import 'path-to-datepicker-package/daterangepicker';
             import 'path-to-datepicker-package/dateRangeTimePicker';
             
             Vue.use(vueCustomElement);
             Vue.customElement('date-picker', {});
             Vue.customElement('date-range-picker', {});
             Vue.customElement('date-range-time-picker', {});

Step 3: Using Components and Displaying Values

In your Vue component, create a data property to hold the selected values and display them in the template. 

        
          <!-- MyComponent.vue -->
          <template>
          <div>
          <!-- Single Date Picker -->
          <date-picker
                aria-label="Select Date"
                @date-updated="handleDateUpdate"
                ></date-picker>
              <p>Selected Date: {{ selectedDate }}</p>
          
              <!-- Date Range Picker -->
              <date-range-picker
                aria-label="Select Date Range"
                @range-updated="handleRangeUpdate"
              ></date-range-picker>
              <p>Selected Date Range: {{ selectedRange }}</p>
          
              <!-- Date Range Time Picker -->
              <date-range-time-picker
                aria-label="Select Date and Time Range"
                @date-time-updated="handleDateTimeUpdate"
                ></date-range-time-picker>
              <p>Selected Date & Time Range: {{ selectedDateTimeRange }}</p>
              </div>
              </template>
          
              <script>
          export default {
            data() {
              return {
                selectedDate: '',               // Single date selection
                selectedRange: '',               // Date range selection
                selectedDateTimeRange: '',       // Date-time range selection
              };
            },
            methods: {
              handleDateUpdate(event) {
                this.selectedDate = event.detail.value;           // Capture as single string
              },
              handleRangeUpdate(event) {
                this.selectedRange = event.detail.value;          // Capture as single string
              },
              handleDateTimeUpdate(event) {
                this.selectedDateTimeRange = event.detail.value;  // Capture as single string
              },
            },
          };
          </script>
React

Step 1: Installation

Install the necessary packages.

Step 2: Create React Wrappers

Define wrappers for each Lit component as before.

Step 3: Using Components and Displaying Values

In the main React component, add state variables to store the selected values and display them.

          
            import React, { useState } from 'react';
            import { DatePicker, DateRangePicker, DateRangeTimePicker } from './DatePickerWrapper';
            
            function App() {
              const [selectedDate, setSelectedDate] = useState('');               // Single date selection
              const [selectedRange, setSelectedRange] = useState('');              // Date range selection
              const [selectedDateTimeRange, setSelectedDateTimeRange] = useState(''); // Date-time range selection
            
              const handleDateUpdate = (e) => {
                setSelectedDate(e.detail.value);           // Capture as single string
              };
            
              const handleRangeUpdate = (e) => {
                setSelectedRange(e.detail.value);          // Capture as single string
              };
            
              const handleDateTimeUpdate = (e) => {
                setSelectedDateTimeRange(e.detail.value);  // Capture as single string
              };
            
              return (
                <div>
                <DatePicker aria-label="Select Date" onDateUpdated={handleDateUpdate} />
                <p>Selected Date: {selectedDate}</p>
            
                <DateRangePicker aria-label="Select Date Range" onRangeUpdated={handleRangeUpdate} />
                <p>Selected Date Range: {selectedRange}</p>
            
                <DateRangeTimePicker aria-label="Select Date and Time Range" onDateTimeUpdated={handleDateTimeUpdate} />
                <p>Selected Date & Time Range: {selectedDateTimeRange}</p>
                </div>
              );
            }
            
            export default App;
Angular

Step 1: Installation

Install the necessary packages.

Step 2: Register Components in app.module.ts

Enable CUSTOM_ELEMENTS_SCHEMA and import the components.

          
            import { NgModule, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';
            import { BrowserModule } from '@angular/platform-browser';
            import { AppComponent } from './app.component';
            
            import 'path-to-datepicker-package/datepicker';
            import 'path-to-datepicker-package/daterangepicker';
            import 'path-to-datepicker-package/daterangetimepicker';
            
            @NgModule({
              declarations: [AppComponent],
              imports: [BrowserModule],
              schemas: [CUSTOM_ELEMENTS_SCHEMA],
              bootstrap: [AppComponent]
            })
            export class AppModule {}

Step 3: Using Components and Displaying Values in Angular

In app.component.html, add the custom elements and display selected values.

        
          // html
          <div>
          <!-- Single Date Picker -->
          <date-picker aria-label="Select Date" (date-updated)="onDateUpdate($event)"></date-picker>
          <p>Selected Date: {{ selectedDate }}</p>
          
          <!-- Date Range Picker -->
          <date-range-picker aria-label="Select Date Range" (range-updated)="onRangeUpdate($event)"></date-range-picker>
          <p>Selected Date Range: {{ selectedRange }}</p>
          
          <!-- Date Range Time Picker -->
          <date-range-time-picker aria-label="Select Date and Time Range" (date-time-updated)="onDateTimeUpdate($event)"></date-range-time-picker>
          <p>Selected Date & Time Range: {{ selectedDateTimeRange }}</p>
          </div>

In app.component.ts, store each value as a single string.

          
            import { Component } from '@angular/core';

            @Component({
              selector: 'app-root',
              templateUrl: './app.component.html',
              styleUrls: ['./app.component.css']
            })
            export class AppComponent {
              selectedDate = '';               // Single date selection
              selectedRange = '';              // Date range selection
              selectedDateTimeRange = '';      // Date-time range selection

              onDateUpdate(event: any) {
                this.selectedDate = event.detail.value;           // Capture as single string
              }

              onRangeUpdate(event: any) {
                this.selectedRange = event.detail.value;          // Capture as single string
              }

              onDateTimeUpdate(event: any) {
                this.selectedDateTimeRange = event.detail.value;  // Capture as single string
              }
            }

Summary

These examples assume that you've configured your framework project to work with web components and that you've imported the Lit Element component correctly. Adjust import paths accordingly.

The interoperability is possible because Web Components, once defined and registered as custom elements, act like native HTML elements that these frameworks can understand and interact with.
